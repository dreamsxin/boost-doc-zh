boost::function — 一个可以用于回调或包装函数对象的泛化的函数指针。

Description（说明）

类模板 function 是包围在带编号的类模板 function0，function1 等外面的一层薄薄的包装。它接受一个带有 N 个参数的函数类型，并从被它所接收的参数实例化的 functionN 继承。

在类模板 function 上的所有操作的语义都等价于在下层的 functionN 对象上的同样操作，尽管需要额外的成员函数以允许 function 对象的特有的拷贝构造和拷贝赋值。

function construct/copy/destruct（function 构造/拷贝/析构）

  1. function();
    后置条件： this->empty()
    抛出： 不会抛出异常。 
  2. function(const functionN& f);
    后置条件： 如果 f 有一个目标，则包含 f 的目标的一个拷贝，或者，如果 f.empty()，则为空。
    抛出： 不会抛出异常，除非拷贝 f 的目标时抛出。 
  3. function(const function& f);
    后置条件： 如果 f 有一个目标，则包含 f 的目标的一个拷贝，或者，如果 f.empty()，则为空。
    抛出： 不会抛出异常，除非拷贝 f 的目标时抛出。 
  4. template<typename F> function(F f);
    条件： F 是一个可由 this 调用的函数对象。
    后置条件： 如果 f 非空，则 *this 以 f 的一个拷贝为目标，或者，如果 f 为空，则 this->empty()。 
  5. function& operator=(const function& f);

    （此处原文如此，怀疑应为 function& operator=(const functionN& f);——译者注）

    后置条件： 如果拷贝构造不抛出异常，如果 f 有一个目标，则 *this 以 f 的目标的拷贝为目标，或者，如果 f.empty()，则为空。如果拷贝构造抛出异常，则 this->empty()。 
  6. function& operator=(const function& f);
    后置条件： 如果拷贝构造不抛出异常，如果 f 有一个目标，则 *this 以 f 的目标的拷贝为目标，或者，如果 f.empty()，则为空。如果拷贝构造抛出异常，则 this->empty()。
    抛出： 当 f 的目标是一个 stateless 函数对象或者是一个引向函数对象的引用，则不抛出异常。 
  7. ~function();
    作用： 如果 !this->empty()，销毁 this 的目标。

function modifiers（function 修改者）

  1. void swap(const function& f);
    作用： 交换 *this 的目标和 f。 
  2. void clear();
    后置条件： this->empty()
    抛出： 不会抛出异常。

function capacity（function 容量）

  1. bool empty() const;
    返回： 如果 this 有一个目标，则为 false，否则，为 true。
    抛出： 不会抛出异常。 
  2. operator safe_bool() const;
    返回： 当 this->empty() 时，为一个在布尔上下文中求值为 false 的 safe_bool，否则，为求值为 true 的 safe_bool。
    抛出： 不会抛出异常。 
  3. bool operator!() const;
    返回： this->empty()
    抛出： 不会抛出异常。

function target access（function 目标访问）

  1. template<typename Functor> Functor* target();
    template<typename Functor> const Functor* target() const;
    返回： 如果 this 存储一个类型为 Functor 的目标，则返回目标的地址。否则，返回 NULL 指针。
    抛出： 不会抛出异常。 
  2. template<typename Functor> bool contains(const Functor& f) const;
    返回： 如果 this->target<Functor>() 为非 NULL，而且 function_equal(*(this->target<Functor>()), f)，则为 true。 
  3. const std::type_info& target_type() const;
    返回： target function object（目标函数对象）的 typeid，或者，如果 this->empty()，则为 typeid(void)。
    抛出： 不会抛出异常。

function invocation（function 调用）

  1. result_type operator()(arg1_type a1, arg2_type a2, ..., argN_type aN) const;
    作用： 当 f 是 *this 的目标时，为 f(a1, a2, ..., aN)。
    返回： 如果 R 为 void，什么都不返回，否则，为调用 f 时的返回值。
    抛出： 如果 this->empty()，则抛出 bad_function_call。否则，为目标函数 f 所可能抛出的任何异常。

function specialized algorithms（function 特有算法）

  1. template<typename Signature, typename Allocator> 
      void swap(function<Signature, Allocator>& f1, 
                function<Signature, Allocator>& f2);
    作用： f1.swap(f2)

    返回： 当 f 存储了一个类型为 Functor 的对象，而且满足下列条件之一时，为 true：
    • g 的类型是 reference_wrapper<Functor>，而且，f.target<Functor>() == g.get_pointer()。
    • g 的类型不是 reference_wrapper<Functor>，而且，function_equal(*(f.target<Functor>()), g)。
    注意： function 对象不是 EqualityComparable（可等值比较） 的。
    基本原理： safe_bool 转换打开一个漏洞，使得两个 function 实例可以通过 == 进行比较，尽管对于实现来说这是不可行的。未定义的 void operator== 封闭这个漏洞并确保一个编译时或连接时错误。/td> 

  2. template<typename Signature, typename Allocator, typename Functor> 
      bool operator!=(const function<Signature, Allocator>& f, Functor g);
    template<typename Signature, typename Allocator, typename Functor> 
      bool operator!=(Functor g, const function<Signature, Allocator>& f);
    template<typename Signature, typename Allocator, typename Functor> 
      bool operator!=(const function<Signature, Allocator>& f, 
                      reference_wrapper<Functor> g);
    template<typename Signature, typename Allocator, typename Functor> 
      bool operator!=(reference_wrapper<Functor> g, 
                      const function<Signature, Allocator>& f);
    template<typename Signature1, typename Allocator1, typename Signature2, 
             typename Allocator2> 
      void operator!=(const function<Signature1, Allocator1>& f1, 
                      const function<Signature2, Allocator2>& f2);
    返回： 当 f 没有存储一个类型为 Functor 的对象时或者它存储一个类型为 Functor 的对象并满足下列条件之一时，为 true：
    • g 的类型是 reference_wrapper<Functor>，而且 f.target<Functor>() != g.get_pointer()。
    • g 的类型不是 reference_wrapper<Functor>，而且 !function_equal(*(f.target<Functor>()), g)。
    注意： function 对象不是 EqualityComparable（可等值比较） 的。
    基本原理： safe_bool 转换打开一个漏洞，使得两个 functionN 实例可以通过 != 进行比较，尽管对于实现来说这是不可行的。未定义的 void operator!= 封闭这个漏洞并确保一个编译时或连接时错误。

Struct template or_

boost::proto::or_ — For matching one of a set of alternate grammars. Alternates are tried in order to avoid ambiguity. When used as a transform, proto::or_<> applies the transform associated with the first grammar that matches the expression.
boost::proto::or_ — 用于匹配一组候选语法中的一个。为避免歧义，候选语法是按顺序尝试的。当被用作变换时，proto::or_<> 将应用与第一个匹配该表达式的语法相对应的变换。

Synopsis

template<typename... G> 
struct or_ :  proto::transform<or_<G...> > {
  // types
  typedef or_ proto_base_expr;
  template<typename Expr, typename State, typename Data> 
  struct impl :  proto::transform_impl< Expr, State, Data > {
    // types
    typedef unspecified result_type;

    // public member functions
    result_type operator()(typename impl::expr_param, 
                           typename impl::state_param, 
                           typename impl::data_param) const;
  };
};

Description

An expression type E matches proto::or_<G0,G1,...Gn> if E matches any Gx for x in [0,n]. 
一个表达式类型 E 匹配 proto::or_<G0,G1,...Gn> 如果 E 匹配任意一个 Gx 其中 x 在[0,n]之中。

When applying proto::or_<G0,G1,...Gn> as a transform with an expression e of type E, state s and data v, it is equivalent to Gx()(e, s, v), where x is the lowest number such that proto::matches<E, Gx>::value is true.
把 proto::or_<G0,G1,...Gn> 作为一个变换应用于一个类型为 E 的表达式 e，状态 s 和数据 v 时，它等价于 Gx()(e, s, v)，其中 x 为满足 proto::matches<E, Gx>::value 为 true 的最小数。

The maximun number of template arguments proto::or_<> accepts is controlled by the BOOST_PROTO_MAX_LOGICAL_ARITY macro.
proto::or_<> 所接受的模板参数的最大数量由 BOOST_PROTO_MAX_LOGICAL_ARITY 宏控制。

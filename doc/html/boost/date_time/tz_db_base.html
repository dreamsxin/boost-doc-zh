Description

该 csv 文件包含了被 tz_db_base 使用的 zone_specs，文件被设计为可以由库的用户定制。在定制该文件(或创建你自己的文件)时，必须遵循特定的格式。

第一行应该包含列头且不被 tz_db_base 处理。

每个记录(行)必须有11个字段。其中一些字段可以为空。每个字段(即使为空)都必须用双引号括起。如： 

 "America/Phoenix" <- 带双引号的字符串
 ""                <- 空字段

有些字段表示一段时间长度。这些字段的格式必须是： 

 "{+|-}hh:mm[:ss]" <- 时长的格式

其中加号或头号是必须的，而秒数是可选的。

由于有的时区不使用夏时制，所以不需要 zone_spec 中的每个字段都包含有值。所有 zone_specs 必须至少有 ID 和 GMT 偏移量。使用夏时制的时区必须填充除了 STD ABBR, STD NAME, DST NAME 以外的所有字段。你要注意，DST ABBR 对于使用夏时制的时区是必须的(更多细节请见字段说明)。

******* 字段及其说明/细节 *********

ID: 包含 zone_spec 的标识字符串。任意唯一的字符串均可。不能有两个 ID 相同。

STD ABBR: STD NAME: DST ABBR: DST NAME: 这四个字段为描述时区用的名字和缩写。虽然这些字段可以是任意字符串，但还是应该小心。这些字段保存的字符串将被用于多个 local_time 的输出。例如： 

 time_zone nyc = tz_db.time_zone_from_region("America/New_York");
 local_time ny_time(date(2004, Aug, 30), IS_DST, nyc);
 cout << ny_time.to_long_string() << endl;
 // 2004-Aug-30 00:00:00 Eastern Daylight Time
 cout << ny_time.to_short_string() << endl;
 // 2004-Aug-30 00:00:00 EDT

注意：精确的格式/函数名可能不同 - 更多细节请见 local_time 的文档。

GMT 偏移量：这是一个小时数，与 utc 相加可得到未进行夏时制调整之前的本地时间。例子如下：America/New_York 偏移 -5 小时，Africa/Cairo 偏移 +2 小时。该字段的格式必须遵循前面所说的时长格式。

DST 调整量：在夏时制生效时加到 gmt_offset 上的小时数。它的格式必须遵循前面所说的时长格式。

DST 开始日期规则：这是一个特殊格式的字符串，表示发生夏时制转换那天是年份中的哪天。它本身由三个字段组成，以分号分隔。第一个字段表示在月份中第 "nth" 个周日。可能的值有：1 (first), 2 (second), 3 (third), 4 (fourth), 5 (fifth), 和 -1 (last)。第二个字段以 0-6 表示周中的星期几(Sun=0)。第三个字段以 1-12 表示月份(Jan=1)。

例如："-1;5;9"="九月最后一个星期五"，"2;1;3"="三月的第二个星期一"

开始时间：开始时间是在转换当天发生转换的时间，以午夜后的小时数来表示。更简单地讲，就是进行转换的时间(以24小时格式表示)。该格式必须遵循前面所说的时长格式，而且它必须为正。

DST 结束日期规则：请见 DST 开始日期规则。不同的是，这个字段是夏时制结束(转换为 STD)的日子。

结束时间：与开始时间相同。

tz_db_base public member functions

  1. void load_from_file(const std::string & pathspec) ;

    可能抛出 data_not_accessible, 或 bad_field_count 异常

  2. bool add_record(const string_type & region, 
                    boost::shared_ptr< time_zone_base_type > tz) ;

    接受一个形如 "America/Phoenix" 的 id 字符，以及该区域的一个 time_zone 对象。字符串 id 必须是唯一的名字，未曾在数据库中存在。

    time_zone_from_region(const string_type & region) const;

    返回一个构建自给定区域规格的 time_zone 对象。如果该区域不存在，则抛出一个 local_time::record_not_found 异常。

  4. std::vector< std::string > region_list() const;

    接受 csv 中的一行，将它转换为 time_zone_type 并将它加入到 map. csv 文件中的 Zone_specs 应该有11个字段来描述该时区。如果 zone_spec 被成功加入到数据库中，则返回 true.

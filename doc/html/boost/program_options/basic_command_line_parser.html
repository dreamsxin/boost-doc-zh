说明

命令行分析器。

该类允许你指定分析所需的所有信息并对命令行进行分析。它主要用于模拟命名函数参数 -- 一个带有5个参数的普通函数是很难使用的，而创建一些较少参数的重载又会引起混淆。

多数情况下，函数 parse_command_line 是更好的选择。

这里有两个 typedefs -- command_line_parser 和 wcommand_line_parser，分别对应 charT == char 和 charT == wchar_t 的情形。

basic_command_line_parser 公有构造/复制/析构

  1. basic_command_line_parser(const std::vector< std::basic_string< charT > > & args);

    以指定参数列表创建一个命令行分析器。'args' 参数不包含程序名。 

  2. basic_command_line_parser(int argc, charT * argv);

    以指定参数列表创建一个命令行分析器。参数与传递给 'main' 的参数相同。

basic_command_line_parser 公有成员函数

    设置使用的选项描述。 

  2. basic_command_line_parser &

    设置使用的位置选项描述。 

  3. basic_command_line_parser & style(int) ;

    设置命令行风格。 

  4. basic_command_line_parser & extra_parser(ext_parser) ;

    设置额外的分析器。 

  5. basic_parsed_options< charT > run() ;

    分析选项并返回分析的结果。错误时抛出。 

  6. basic_command_line_parser & allow_unregistered() ;

    指定允许未登记的选项并跳过它。对于每一个看起来象选项但又不能识别的命令 token，将增加一个 basic_option<charT> 实例到结果中，并将 'unrecognized' 字段设置为 'true'。可以通过 'collect_unrecognized' 函数收集所有未识别选项。

类型 style_t

boost::program_options::command_line_style::style_t

摘要

enum style_t { allow_long =  1, allow_short =  allow_long << 1, 
               allow_dash_for_short =  allow_short << 1, 
               allow_slash_for_short =  allow_dash_for_short << 1, 
               long_allow_adjacent =  allow_slash_for_short << 1, 
               long_allow_next =  long_allow_adjacent << 1, 
               short_allow_adjacent =  long_allow_next << 1, 
               short_allow_next =  short_allow_adjacent << 1, 
               allow_sticky =  short_allow_next << 1, 
               allow_guessing =  allow_sticky << 1, 
               case_insensitive =  allow_guessing << 1, 
               allow_long_disguise =  case_insensitive << 1, 
               unix_style =  (allow_short | short_allow_adjacent | short_allow_next
                      | allow_long | long_allow_adjacent | long_allow_next
                      | allow_sticky | allow_guessing 
                      | allow_dash_for_short), 
               default_style =  unix_style };

说明

选项的各种可能风格。

选项有各种风格，有"长"选项，它们以 "--" 开始，有"短"选项，以 "-" 或 "/" 开始。两种风格都可以被允许或被禁止，参见 allow_long 和 allow_short. 短选项中所允许使用的字符也是可配置的。

选项的值可以在同一个 token 中指定，如 ("--foo=bar")，也可以在下一个 token 中给出。

可以用相同的字符来引入长选项和短选项，参见 allow_long_disguise.

最后，还支持猜测(仅指定选项的前缀)及大小写不敏感的处理。

allow_long
允许 "--long_name" 风格。
allow_short
允许 "-<single character" 风格。
allow_dash_for_short
允许短选项使用 "-"。
allow_slash_for_short
允许短选项使用 "/"。
long_allow_adjacent

允许长选项中的选项参数在同一个 token，如 --foo=10

long_allow_next

允许长选项中的选项参数在下一个 token。

short_allow_adjacent

允许短选项中的选项参数在同一个 token。

short_allow_next

允许短选项中的选项参数在下一个 token。

allow_sticky

允许合并多个短选项，如将 "-s -k" 写为 "-sk"。除最后一个选项外，其它所有选项不可接受参数。例如，如果 "-s" 要接受一个参数，则 "k" 将被视为参数而不是另一个选项。Dos风格的短选项不可以合并。

allow_guessing

允许对长选项使用简写，如果简写可以明确标识一个长选项。如果使用猜测功能，则不可以有某个长选项名是另一个长选项名的前缀。

case_insensitive

忽略选项的大小写。待完成。

这一点是否仅适用于长选项？

allow_long_disguise

允许长选项带有单个选项开始字符，如 -foo=10

unix_style

或多或少的传统 unix 风格。

default_style

缺省风格。

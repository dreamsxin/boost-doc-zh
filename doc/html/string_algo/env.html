构建

整个库以头文件方式提供。但是，某些算法的 Regex 变体依赖于 Boost.Regex 库。所有这样的算法都隔离在 boost/algorithm/string_regex.hpp 中。如果使用了这个头文件，应用程序必须连接 Boost.Regex 库。

示例

展示本库基本用法的示例放在 libs/algorithm/string/example 目录中。本库的每一个部分有一个单独的文件。请按照 boost 构建指南使用 bjam 来构建这些示例。若要成功构建 regex 示例，就需要 Boost.Regex 库。

测试

本库的一整套测试用例位于 libs/algorithm/string/test 目录。测试用例可以用 boost 构建系统执行。对于算法的的正则表达式变体的测试，需要 Boost.Regex 库。

可移植性

本库在下列编译器中成功编译并测试：

  • Microsoft Visual C++ 7.0
  • Microsoft Visual C++ 7.1
  • GCC 3.2
  • GCC 3.3.1

关于特定编译器的更多信息，请参见 Boost regression tables。

在不支持模板偏特化的平台上有已知的限制。本库还依赖正确地实现 std::iterator_traits 类。如果一个编译器附带的标准库有问题，string 算法库就不能完整地实现其功能。通常这意味着简单的指针迭代器不能和本库函数一起工作。

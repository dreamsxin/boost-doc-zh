Installing the library 库的安装

这个库仅由 include 文件组成，因此不需要安装过程。boost 的 include 目录必须加入到 include 路径中。有许多 include 文件提供了不同的功能：

  • lambda/lambda.hpp 为不同的 C++ 操作符定义了 lambda 表达式，参见 “Operator expressions” 部分。

  • lambda/bind.hpp 为最多 9 个参数定义了 bind 表达式，参见 “Bind expressions” 部分。

  • lambda/if.hpp 定义了相当于 if 语句和条件操作符的 lambda 函数，参见 “Lambda expressions for control structures” 部分（包括 lambda.hpp）。

  • lambda/loops.hpp 定义了相当于循环结构的 lambda 函数，参见 “Lambda expressions for control structures” 部分。

  • lambda/switch.hpp 定义了相当于 switch 语句的 lambda 函数，参见 “Lambda expressions for control structures” 部分。

  • lambda/construct.hpp 提供了用于写带有构造函数，析构函数，new 和 delete 调用的 lambda 表达式的工具，参见 “Construction and destruction” 部分（包括 lambda.hpp）。

  • lambda/casts.hpp 提供了各种强制转型以及 sizeof 和 typeid 的 lambda 版本，参见 “Cast expressions” 部分。

  • lambda/exceptions.hpp 给出了在 lambda 函数中抛出和捕获异常的工具，参见 “Exceptions” 部分（包括 lambda.hpp）。

  • lambda/algorithm.hpp 和 lambda/numeric.hpp（与标准 algorithm 和 numeric 头文件相对）允许嵌入 STL 算法调用，参见 “Nesting STL algorithm invocations” 部分。

其它一些包中的头文件是内部使用的。另外，这个库依赖于另外两个 Boost 库，Tuple [tuple] 和 type_traits [type_traits] 库，还有 boost/ref.hpp 头文件。

所有的定义都位于名字空间 boost::lambda 和它的子名字空间中。

Conventions used in this document 此文档中用到的约定

在大多数代码示例中，我们省略了那些位于名字空间 std 和 boost::lambda 中的名字的名字空间前缀。缺省 using 声明

using namespace std;
using namespace boost::lambda;

被假定为有效。

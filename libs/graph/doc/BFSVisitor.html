BFS Visitor Concept 广度优先遍历器概念

这个概念为 breadth_first_search() 定义了遍历器接口。用户可以定义一个具有BFS遍历器接口的类，并将该类的一个对象传递给 breadth_first_search()，从而在图搜索期间增加一些动作。

Refinement of 精化自

可复制构造 Copy Constructible (遍历器的复制应该是一个轻量级的操作)。

Notation 记号

V 一个以BFS遍历器为模的类型。
vis 一个类型为 V 的对象。
G 一个以 Graph 为模的类型。
g 一个类型为 G 的对象。
e 一个类型为 boost::graph_traits<G>::edge_descriptor 的对象。
s,u 一个类型为 boost::graph_traits<G>::vertex_descriptor 的对象。

Associated Types 关联类型

无

Valid Expressions 有效表达式

NameExpressionReturn TypeDescription
顶点初始化 vis.initialize_vertex(s, g) void 在开始图搜索之前，对图中的每个点调用。
发现顶点 vis.discover_vertex(u, g) void 当首次遇到某个顶点时调用。
顶点检查 vis.examine_vertex(u, g) void 当某个顶点从队列中弹出时调用。在对顶点 u 的每条出边调用 examine_edge() 前发生。
边检查 vis.examine_edge(e, g) void 在每个顶点被发现后，对其每条出边调用。
树边 vis.tree_edge(e, g) void 在某条边成为搜索树的边成员时调用。
非树边 vis.non_tree_edge(e, g) void 对有向图的反向边或交叉边以及无向图的交叉边调用。
灰色目标 vis.gray_target(e, g) void 对非树边的一个子集调用，该子集由检查时目标顶点为灰色的边组成。灰色表示该顶点当前在队列中。
黑色目标 vis.black_target(e, g) void 对非树边的一个子集调用，该子集由检查时目标顶点为黑色的边组成。黑色表示该顶点已从队列中移除。
结束顶点 vis.finish_vertex(u, g) void 在某个顶点的所有出边被加至搜索树且所有邻接顶点被发现后(但在邻接顶点的出边被检查之前)对该顶点调用。

Models 模型

  • bfs_visitor

Python

要在Python中实现一个BFSVisitor概念的模型，应创建一个从图的 BFSVisitor 类型(名为 GraphType.BFSVisitor)派生的新类。事件和语法与C++中的遍历器相同。以下是一个 Python bgl.Graph 图类型的例子： 
class count_tree_edges_bfs_visitor(bgl.Graph.BFSVisitor):
  def __init__(self, name_map):
    bgl.Graph.BFSVisitor.__init__(self)
    self.name_map = name_map

  def tree_edge(self, e, g):
    (u, v) = (g.source(e), g.target(e))
    print "Tree edge ",
    print self.name_map[u],
    print " -> ",
    print self.name_map[v]

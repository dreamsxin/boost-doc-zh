已知问题和改进表(Known Issues, and Todo List)

这部分列举一些已知的问题。

占主导地位的(Predominantly)是一个 TODO 列表，或者一个将来可能增强的功能列表。标记为"(高优先级)High Priority"的问题影响( effect )这个组件(compement)的合适的功能(proper functioning)，并且应当尽快地修正。标记为"中优先级(Medium Priority)"是预期的功能增强，经常是关于这个组件的性能，但不影响精度的功能。标记为"低优先级(Low Priority)"的问题或许在某个时间应当进行调查研究。 这种分类明显是很主观的。

如果在这里你没有看到某个组件列举在这里，那是因为我们不知道这个组件的任何问题。

tgamma
  • 兰克泽斯逼近(Lanczos approximation) 可以被进一步优化吗? (低优先级)
Incomplete Beta
  • 研究 Didonato 和 Morris 的对于数值很大的a和b进行的渐近展开(asymptotic expansion)(中优先级).
Inverse Gamma
  • 研究如果第一个逼近足够好的话，那么我们是否能够跳过所有的其它的迭代。(中优先级).
Polynomials
  • Legendre 和 Laguerre 多项式在不同的平台上有明显不同的误差率，考虑它们使用基本的算术运算。这可能告诉我们某些事情，也可能没有。 (低优先级).
Elliptic Integrals
  • Carlson 的算法基本上是从 Xiaogang Zhang 的 Google Summer of Code student project中没有改变地引进进来，并基于Carlson 的原始论文。然而，自从那时起，Carlson 已经修订了他的算法(参考椭圆积分中的参考文档了解更多信息)不改变精度和精确性，我们可能也可以利用这些改进 (低优先级)。

  • 当参数非常大的时候，Carlson算法 (主要是 RJ)易于发生内部的向上溢出/向下溢出。齐性关系(homogeneity relation) ：

    RF(ka, kb, kc) = k-1/2 RF(a, b, c)

    以及

    RJ(ka, kb, kc, kr) = k-3/2 RJ(a, b, c, r)

    可以用来回避这些问题：假定问题的值域可以被精确地定义。(中优先级)

  • Carlson 算法中的RC 可以被简化为元函数( elementary funtions )(asin 和 log)，这种方式而不是Carlson 的算法会更高效一些吗？(低优先级)。
  • 我们应当实现 Carlson 算法的RG吗? 它没有被 Legendre 形式积分要求，但是有一些人发现这个值很有用。(低优先级)。
  • 还有其它的许多积分 D(φ, k), Z(β, k), Λ0(β, k) 以及 Bulirsch's el functions 可以使用Carlson的积分来实现。(低优先级)。
  • 积分K(k) 和 E(k) 可以使用有理逼近来实现(出于效率和精度的考虑)，假定我们可以找到些逼近方法。 (中优先级)。
  • ellint_3 有一个子定义域(sub-domain)没有实现 (参考相关文档了解更多信息)，现在还不知道如何解决这个问题，或者在实际中它是否是一个问题－尤其是大多数的其它的库的实现也不支持这个定义域。(中优先级)
Inverse Hyperbolic Functions
  • 在函数log1p 开始广泛可用之前，这些函数从先前的Boost版本中继承过来。 使用这些函数可以使它们实现得更好吗？这可能仅对高精度的类型才是一个问题。(低优先级)。
Statistical distributions
  • 对于非常大的自由度，学生t分布或许可以调整为一个正态分布来作为一个更好的逼近??

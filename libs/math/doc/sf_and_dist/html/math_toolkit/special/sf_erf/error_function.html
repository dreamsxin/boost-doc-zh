#include <boost/math/special_functions/erf.hpp>


namespace boost{ namespace math{

template <class T>
calculated-result-type erf(T z);

calculated-result-type erf(T z, const Policy&);

calculated-result-type erfc(T z);

template <class T, class Policy>
calculated-result-type erfc(T z, const Policy&);

}} // namespaces

函数返回值的类型使用返回值推导法则:来确定，当x是整型的时候，返回值的类型是double,否则返回值是类型是T.

最后一个策略 参数是可选的并且可以用来控制函数的行为: 如何处理错误, 使用哪种层次的精度等等. 参见策略文档了解更多信息。

测试

这些函数的测试包含两部分: 基本的合理性测试使用Mathworld's 在线求值产生的抽样数据，而精度测试的数据使用精度为1000-bits的NTL::RR 库以及这个库的实现生成. 注意：这些函数的一般版本和类型特定的版本在内部使用不同的实现，所以这就可以给我们生成相互独立的测试数据。使用我们的测试数据来测试其它的"已知的良好 (known good)" 库同样也提供了一个合理性测试.

实现

所有的这些实现都首先使用通常的反射方程来使得它们的参数为正数: 

erf(-z) = 1 - erf(z);

erfc(-z) = 2 - erfc(z);  // 当 -z < -0.5时作为选择

erfc(-z) = 1 + erf(z);   // 当 -0.5 <= -z < 0时作为选择

这些函数的一般版本使用不完全γ函数来实现.

当小数 (尾数) 大小是已知的时候(当前针对于 53, 64 和 113-bit 实数, 加上提升为double的单精度的24-bits实数 ) 使用级数的有理逼近由JM发明

对于 z <= 0.5，那么将会使用erf的有理逼近，这一点是基于erf是一个偶函数且可以使用下面的方式来计算的观察: 

erf(z) = z * (C + R(z*z));

其中 ，针对于的R(z*z)有理逼近进行了绝对误差的优化: 一旦它的绝对误差与常量C相比足够小的话，任何在计算R(z*z)的过程中的舍入误差将会在计算结果中消失。因结在这个区域中的erf函数和erfc函数的误差很低：在非常少数的情况下最生一个bit不是正确的.

对于 z > 0.5 ，我们观察到在一个小的区间[a, b) 上对于一些常量 c.: 

erfc(z) * exp(z*z) * z ~ c

因此对于z > 0.5，我们使用下面的方法来计算 erfc: 

erfc(z) = exp(-z*z) * (C + R(z - B)) / z;

R(z - B) 也针对于绝对误差进行了优化, 并且常量C是erfc(z) * exp(z*z) * z 在这个范围的端点处值的平均值。再一次，一旦R(z - B)的绝对误差与常量c相比足够小的话，那么c+R(z-B)将会进行正确的舍入。并且精确性只依赖于函数exp。实际上，在大多数情况下，误差被限制到结果的最后一个bit。选择常量B使得范围的左端的有理逼近为0。

对于区间 [a, +∞]上的较大的z,上面的逼近修改为：

erfc(z) = exp(-z*z) * (C + R(1 / z)) / z;

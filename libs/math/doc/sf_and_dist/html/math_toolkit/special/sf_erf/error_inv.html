误差函数反函数

概要 

#include <boost/math/special_functions/erf.hpp>


namespace boost{ namespace math{

template <class T>
calculated-result-type erf_inv(T p);

template <class T, class Policy>
calculated-result-type erf_inv(T p, const Policy&);

template <class T>
calculated-result-type erfc_inv(T p);

template <class T, class Policy>
calculated-result-type erfc_inv(T p, const Policy&);

}} // namespaces

函数返回值的类型使用返回值推导法则:来确定，当x是整型的时候，返回值的类型是double,否则返回值是类型是T.

最后一个策略 参数是可选的并且可以用来控制函数的行为: 如何处理错误, 使用哪种层次的精度等等. 参见策略文档了解更多信息。

说明 
template <class T>
calculated-result-type erf_inv(T z);

template <class T, class Policy>
calculated-result-type erf_inv(T z, const Policy&);

返回z的误差函数的反函数 , 也就是变量x的值满足: 

p = erf(x);


template <class T>
calculated-result-type erfc_inv(T z);

template <class T, class Policy>
calculated-result-type erfc_inv(T z, const Policy&);

返回z的误差函数的反函数的补集，也就是变量x的值满足: 

p = erfc(x);

精确度

对于包含以及超过80-bit 的 long double类型所使用的逼近方法的精度略中于 2 epsilon(10的-5次方). 对于精度较高的类型，函数的精度与正向误差函数的精度一样。

测试

有两类测试:

  • 基本的合理性测试试图从x到p的值进行往返测试。. 这些测试具有测试的容许空间：总体上，误差函数及其反函数在某些位置变化如此如之快以至于超过几个有效数字的往返测试都是不可能的。尤其是当p的值非常接近于1的时候：在这种情况下，在我们测试开始的地方没有足够的“信息量”来使得反函数可以回退。
  • 使用高精度的测试数据来进行精度检查.给定精确的输入值的时候， 这些测试用来测量结果的精度.
实现

这些函数使用由 JM发明的有理逼近来计算精度~10-19的结果的初始近似值，然后，仅当对于类型T的精确度与epsilon(10的-5次方)相比不足时，我们使用 Halley 迭代来清除结果。

构造erf/erfc函数的有理逼近出乎意料的困难，尤其是当精度很高时。出于这个原因，并没有尝试使用得128-bit的有理数的计算精度达到10-34。

在下面的讨论中，p是传递给函数erf_inv的值，而q是传递给函数erf_inv的值，满足：p = 1 - q 且 q = 1 - p 在两种情况下我们想要计算出相同的结果x.

对于p < 0.5 ，erf函数的反函数非常平缓，而且下面的逼近: 

x = p(p + 10)(Y + R(p))

对于一个常量Y给出了很好的计算结果, 同时 R(p)对与|Y|相比 的较小的绝对误差进行了优化。

对于 q < 0.5 ，事情就变得很棘手 , 在区间 0.5 > q > 0.25 之上，下面的逼近运行得非常好: 

x = sqrt(-2log(q)) / (Y + R(q))

而勇于进取 q < 0.25, 假定

z = sqrt(-log(q))

那么结果由下面的方程给定: 

x = z(Y + R(z - B))

与前面一样Y是一个常量且有理函数R针对于与|Y|相比的绝对误差进行了优化。B也是一个常量：它是对于每一个逼近都合法的z的最小值。每一种形式都有几个逼近方法，每一种逼近方式都超过了erfc函数的范围(在long double精度上，与double比较的扩充的指数范围意味着超过的范围延续了很长一段距离)。

第一类贝赛尔函数和第二类贝赛尔函数

概要 
template <class T1, class T2>
calculated-result-type cyl_bessel_j(T1 v, T2 x);

template <class T1, class T2, class Policy>
calculated-result-type cyl_bessel_j(T1 v, T2 x, const Policy&);

template <class T1, class T2>
calculated-result-type cyl_neumann(T1 v, T2 x);

template <class T1, class T2, class Policy>
calculated-result-type cyl_neumann(T1 v, T2 x, const Policy&);
说明

函数cyl_bessel_j 和函数cyl_neumann 分别返回第一类贝赛尔函数和第二类贝赛尔函数的结果:

cyl_bessel_j(v, x) = Jv(x)

cyl_neumann(v, x) = Yv(x) = Nv(x)

其中:

当T1和T2是不同类型的时候，这些函数的返回类型使用返回类型推导法则来确定。 这些函数也对T1是一个整数的常见情况进行了优化.

最后一个策略 参数是可选的并且可以用来控制函数的行为: 如何处理错误, 使用哪一种层次的精度等等. 参见策略文档获取更多信息。

当结果是未定义的或是复数的时候返回定义域错误。对于函数cyl_bessel_j 这会在x < 0 且 v 不是一个整数或当x==0且v != 0 的时候发生。对于函数cyl_neumann 这会在 x <= 0 发生。

与 GSL 库和 Cephes 库的比较非常有趣: Cephes 库和这个库都对整数次数的情况做了优化 - 产生很明显的结果 - 对大部分处理使用一般的处理情况就可以获得稍微好的精确性, 就像由GSL库在整数参数的情况下获得的更好的准确性所显示的那样. 这个库的实现趋向于在参数变大的时候获得更好的表现, Cephes 库在一些测试数据中产生了一些明显的不精确的结果 (没有有效数字校正),甚至GSL库对于函数 Jv的一些输入的表现也很差。 注意：通过双重检查这些结果的方式,Cephes 库和 GSL 库的最差运行情况使用functions.wolfram.com进行了重新计算 并且这些结果与我们的测试数据进行了重新比较: 测试数据中没有发现误差.

实现

库的实现大部分是在处理各种特殊情况:

当 x 负数的时候, 次数v 必须是一个整数否则结果是一个定义域错误. 如果次数是一个整数，那么如果次数是奇数则函数是一个奇函数，如果次数是一个偶数则函数是一个偶函数, 这里要求满足x > 0.

注意如果次数是一个整数, 那么这些方程简化为:

J-n = (-1)nJn

Y-n = (-1)nYn

然而, 一般来说, 一个负的次数暗示我们必须计算 J 和 Y.

当x比次数 v 大，那么将会使用 M. Abramowitz 和 I.A. Stegun, 数学函数手册 (Handbook of Mathematical Functions 9.2.19)中的大数x的渐近展开(这比 A&S 9.2.5中的更值得依赖).

当系数 v 是一个整数 这种方法首先依据哪一个更稳定来使用前向循环或后向循环将结果与 J0, J1, Y0 and Y1关联起来(密勒算法). J0, J1, Y0 和 Y1使用对于小数|x| 的有理数极值逼近以及对于大数|x|的渐近展开来计算系数:

以及

J.F. Hart et al, Computer Approximations, John Wiley & Sons, New York, 1968.

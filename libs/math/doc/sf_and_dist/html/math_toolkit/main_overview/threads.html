线程安全性(Thread Safety)

假定这个库中的函数模板和类模板都是使用内建的(built-in)浮点类型(例如，float , double , long double)实例化，那么这个库是完全的线程安全的并且可重入的(re-entrant)。

然而，当与用户自定义的数值类型一起使用时，这个库 不是线程安全的

后面一个限制的理由是使用类似于下面的方式来初始化符号常量：

static const T coefficient_array = { ... list of values ... };

当T是内建的(built-in)浮点类型时，上面的代码永远是线程安全的，但是当T是用户自定义数据类型时，上面的代码就不是线程安全的：因为在这种情况下需要运行T的构造函数，这就导致了一个潜在的竞争条件( race conditions )。

这个限制可能会在以后的版本中得到解决。

Quaternion 非成员运算符

一元 取正 运算符 
template<typename T>	
quaternion<T> operator + (quaternion<T> const & q);

这个一元运算符仅仅是将四元数 q 返回.

一元 取负 运算符 
template<typename T>
quaternion<T> operator - (quaternion<T> const & q);

这个一元运算符返回四元数 q 的相反数.

二元加法运算符 
template<typename T>	quaternion<T>	operator + (T const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator + (quaternion<T> const & lhs, T const & rhs);
template<typename T>	quaternion<T>	operator + (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator + (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	quaternion<T>	operator + (quaternion<T> const & lhs, quaternion<T> const & rhs);

这些加法运算符返回quaternion<T>(lhs) += rhs的结果.

二元减法运算符 
template<typename T>	quaternion<T>	operator - (T const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator - (quaternion<T> const & lhs, T const & rhs);
template<typename T>	quaternion<T>	operator - (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator - (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	quaternion<T>	operator - (quaternion<T> const & lhs, quaternion<T> const & rhs);

这些加法运算符返回 quaternion<T>(lhs) -= rhs的结果.

二元乘法运算符 
template<typename T>	quaternion<T>	operator * (T const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator * (quaternion<T> const & lhs, T const & rhs);
template<typename T>	quaternion<T>	operator * (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator * (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	quaternion<T>	operator * (quaternion<T> const & lhs, quaternion<T> const & rhs);

这些乘法运算符返回 quaternion<T>(lhs) *= rhs的结果.

二元除法运算符 
template<typename T>	quaternion<T>	operator / (T const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator / (quaternion<T> const & lhs, T const & rhs);
template<typename T>	quaternion<T>	operator / (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	quaternion<T>	operator / (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	quaternion<T>	operator / (quaternion<T> const & lhs, quaternion<T> const & rhs);

这些运算符返回 quaternion<T>(lhs) /= rhs的结果. 除以 0 依然是一个错误...

相等(==)运算符 
template<typename T>	bool	operator == (T const & lhs, quaternion<T> const & rhs);
template<typename T>	bool	operator == (quaternion<T> const & lhs, T const & rhs);
template<typename T>	bool	operator == (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	bool	operator == (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	bool	operator == (quaternion<T> const & lhs, quaternion<T> const & rhs);

当且仅当quaternion<T>(lhs) 中的四个成员分别与 quaternion<T>(rhs) 中的四个对应 的成员分别相等这个运算符返回 真. 与任何的浮点数一样, 这本质上是没有意义的. (译注：作者的意思是这样的，从本质上来说，比较两个四元数相等是 没有意义的， 因为，任意一个四元数的表示都是不绝对精确的，正是因为如此， 判断两个不精确的量是否相等 本质上是没有意义的， 这一点类似于我们判断两个浮点数是否相等，因为对于任意一个浮点数， 计算机都不可能 精确的表示出来，这一点由计算机的固有特性决定. 正是因为计算机无法精确的表示出任意一个浮点数， 所以，判断两个浮点数是否相等本质上也是没有意义的)

不相等(!=)运算符 
template<typename T>	bool	operator != (T const & lhs, quaternion<T> const & rhs);
template<typename T>	bool	operator != (quaternion<T> const & lhs, T const & rhs);
template<typename T>	bool	operator != (::std::complex<T> const & lhs, quaternion<T> const & rhs);
template<typename T>	bool	operator != (quaternion<T> const & lhs, ::std::complex<T> const & rhs);
template<typename T>	bool	operator != (quaternion<T> const & lhs, quaternion<T> const & rhs);

这些函数返回真，当且仅当quaternion<T>(lhs) == quaternion<T>(rhs) 为假. 与任何的浮点数一样, 这本质上是没有意义的.

流抽取运算符 
template<typename T, typename charT, class traits>
::std::basic_istream<charT,traits>& operator >> (::std::basic_istream<charT,traits> & is, quaternion<T> & q);

从下面的任意一种形式中抽取一个四元数 q ( a, b, c 和 d 的类型为 T):

a (a), (a,b), (a,b,c), (a,b,c,d) (a,(c)), (a,(c,d)), ((a)), ((a),c), ((a),(c)), ((a),(c,d)), ((a,b)), ((a,b),c), ((a,b),(c)), ((a,b),(c,d))

输入值必须能够转换为T类型的值. 如果遇到了不合法的输入, 将会调用函数 is.setstate(ios::failbit) (该函数将会抛出异常 ios::failure (27.4.5.3)).

返回: is.

这一串可以接受的输入格式的原理在于 或者我们有一个由4个实数组成的实数串 或者 我们有两个复数, 并且在那种情况下可以被格式化为合适的复数, 那么这种输入形式应当被接受. 这样，潜在的二义性就被消除了 (例如 (a,b) 表示 (a,b,0,0) 而不是(a,0,b,0), 也就是说. 这种形式被解析为两个实数组成的串 而不是两个虚部为0的复数组成的串 ).

流插入运算符 
template<typename T, typename charT, class traits>
::std::basic_ostream<charT,traits>& operator << (::std::basic_ostream<charT,traits> & os, quaternion<T> const & q);

将四元数 q 插入到流 os中 就像它被实现为下面这样: 

template<typename T, typename charT, class traits>
::std::basic_ostream<charT,traits>& operator << (	
               ::std::basic_ostream<charT,traits> & os,
               quaternion<T> const & q)
{
   ::std::basic_ostringstream<charT,traits>	s;

   s.flags(os.flags());
   s.imbue(os.getloc());
   s.precision(os.precision());

   s << '(' << q.R_component_1() << ','
            << q.R_component_2() << ','
            << q.R_component_3() << ','
            << q.R_component_4() << ')';

   return os << s.str();
}

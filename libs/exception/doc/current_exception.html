Boost Exception

current_exception

#include <boost/exception_ptr.hpp>

namespace
boost
    {
    exception_ptr current_exception();
    }

要求:

在catch块的外部不能在调用函数 current_exception.

此外，要安全地将一个异常从一个线程复制至另一个线程，如果该异常对象是由 current_exception 或 copy_exception 复制的，则这两份拷贝必须不能用共享的状态。具有值类型语义的异常(以及 boost::exception 类型本身)满足这一要求。

返回值:

  • 异常对象指针 exception_ptr ,指向当前处理的异常对象或者其拷贝.
  • 如果函数需要分配内存,但是分配失败, exception_ptr 对象指向一个 std::bad_alloc 的实例.

异常抛出:

没有异常.

注意:

  • 连续两次的调用函数 current_exception ,并不保证得到的总是同一的异常对象.
  • current_exception 的正确实现可能需要编译器的支持，除非在当前被处理的异常对象传递给 throw 的时候使用了 enable_current_exception。当 current_exception 不能正确复制 current exception 对象时，它将返回一个 exception_ptr, 指向一个类型尽可能接近于原有异常类型的对象，unknown_exception 被作为最终可用的类型。所有这些类型均派生自 boost::exception，且：
    • 如果原有异常对象派生自 boost::exception，则该对象的 boost::exception 子对象被引向所返回的 exception_ptr，由 boost::exception 复制构造函数来初始化；
    • 如果可能，该异常将包含原有异常对象的 std::type_info，可通过 get_error_info<original_exception_type> 访问。

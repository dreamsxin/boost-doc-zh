Boost.Compatibilty 库

本库提供了一些变通的方法，以允许 Boost 的其它库可以在一些不符合标准的平台上使用。我们希望在以后的某个时间，标准库的提供者可以更加符合标准，从而可以删除该库。

缺少了C++标准库CXX头文件(如<cstdio>)的变通方法

Python 脚本: generate_cpp_c_headers.py 用于在某些缺少了 C++ C 头文件(如<cstdio>)的平台上创建一整套头文件。由该脚本创建的头文件位于目录 boost/compatibility/cpp_c_headers 中。要使用这些头文件，请将该目录增加到包含文件搜索路径中。例如：

cxx -I/usr/local/boost/boost/compatibility/cpp_c_headers ...

支持的平台有：

  • Compaq Alpha, RedHat 6.2 Linux, Compaq C++ V6.3 (cxx)
  • Compaq Alpha, Tru64 Unix V5.0, Compaq C++ V6.2 (cxx)
  • Silicon Graphics, IRIX 6.5, MIPSpro Compilers: Version 7.3.1.1m (CC)

除了使用 Boost.Compatibility 库的 CXX 头文件，还有其它功能更强大的选择，如 STLport 或 ISOCXX. 不过，与这些选择相比，脚本 generate_cpp_c_headers.py 是非常轻量级的(只有少于100行的非注释Python代码)，更为专一，也更容易维护，所以更适合用作临时的变通方法。

这是由 Ralf W. Grosse-Kunstleve 贡献的。

缺少了C++标准库 <limits> 头文件的变通方法 boost/limits.hpp

有几个Boost库需要使用标准库的 <limits> 头文件，而不符合标准的编译器和库并不总会提供这个头文件。头文件 boost/limits.hpp 在可能的情况下包含标准库的 <limits> 头文件，否则包含 boost/detail/limits.hpp. boost/config.hpp 中的 BOOST_NO_LIMITS 用于确定 <limits> 是否可用。

请参考测试程序 limits_test.cpp

这是由 Jens Maurer 贡献的。

可读 Property Map

可读 Property Map 通过调用 get() 函数提供了对与某个给定的键相关联的值的读访问。get() 函数返回值对象的拷贝。

强化自

可复制构造

符号

PMap 一个类型，符合可读 Property Map.
pmap 类型 PMap 的一个对象
key 类型 boost::property_traits<PMap>::key_type 的一个对象

关联类型

Value Type boost::property_traits<PMap>::value_type 属性的类型
Reference Type boost::property_traits<PMap>::reference 可转换为值类型的类型
Key Type boost::property_traits<PMap>::key_type 用于查找属性的键对象的类型。property map 可以依照键类型参数化，这种情况下该 typedef 可以为 void.
Property Map Category boost::property_traits<PMap>::category 属性的分类：可转换为 readable_property_map_tag 的类型

概念检查类

  template <class PMap, class Key>
  struct ReadablePropertyMapConcept
  {
    typedef typename property_traits<PMap>::key_type key_type;
    typedef typename property_traits<PMap>::category Category;
    typedef boost::readable_property_map_tag ReadableTag;
    void constraints() {
      function_requires< ConvertibleConcept<Category, ReadableTag> >();

      val = get(pmap, k);
    }
    PMap pmap;
    Key k;
    typename property_traits<PMap>::value_type val;
  };

设计说明

曾经考虑过将该概念命名为 "只读"。但是，这个名字不太适合，因为概念天然上是叠加的，而不是相减的。当我们定义读写 Property Map 时这将变得更为明显，它可以从可读 Property Map 和可写 Property Map 概念强化而来，而不是通过组合 "只读" 和 "只写" 概念而得到！

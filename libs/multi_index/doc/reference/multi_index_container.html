类模板 multi_index_container

这是Boost.MultiIndex的主要组成部分。multi_index_container 是一个容器类模板，持有编译期用户定义的indices列表。这些索引为管理multi_index_container中的元素提供了不同的接口。而multi_index_container本身仅提供基本的构造功能以及对内部索引的访问。

multi_index_container 类型以其内部元素的类型及一个指明索引的非空MPL Forward Sequence来实例化。

为了便于使用，multi_index_container继承了其第一个索引的所有公有办法及类型。也包括了与该索引相关的全局操作符和全局函数(如比较和 swap)。 

template<
  typename Value,
  typename IndexSpecifierList=indexed_by<ordered_unique<identity<Value> > >,
  typename Allocator=std::allocator<Value> >
class multi_index_container
{
public:

  // 类型:

  typedef implementation defined   ctor_args_list;
  typedef implementation defined   index_specifier_type_list;
  typedef implementation defined   index_type_list;
  typedef implementation defined   iterator_type_list;
  typedef implementation defined   const_iterator_type_list;
  typedef Allocator                allocator_type;

  // 嵌套类模板:

  template<int N>
  struct nth_index                {typedef implementation defined type;};
  template<typename Tag>
  struct index                    {typedef implementation defined type;};
  template<int N>
  struct nth_index_iterator       {typedef implementation defined type;};
  template<int N>
  struct nth_index_const_iterator {typedef implementation defined type;};
  template<typename Tag>
  struct index_iterator           {typedef implementation defined type;};
  template<typename Tag>
  struct index_const_iterator     {typedef implementation defined type;};

  // 构造/复制/析构:

  explicit multi_index_container(
    const ctor_args_list& args_list=ctor_args_list(),
    const allocator_type& al=allocator_type());
  template<typename InputIterator>
  multi_index_container(
    InputIterator first,InputIterator last,
    const ctor_args_list& args_list=ctor_args_list(),
    const allocator_type& al=allocator_type());
  multi_index_container(
    const multi_index_container<Value,IndexSpecifierList,Allocator>& x);

  ~multi_index_container();

  multi_index_container<Value,IndexSpecifierList,Allocator>& operator=(
    const multi_index_container<Value,IndexSpecifierList,Allocator>& x);

  allocator_type get_allocator()const;

  // 索引取出

  template<int N> typename nth_index<N>::type& get();
  template<int N> const typename nth_index<N>::type& get()const;
  template<typename Tag> typename index<Tag>::type& get()
  template<typename Tag> const typename index<Tag>::type& get()const;

  // 迭代器投影

  template<int N,typename IteratorType>
    typename nth_index_iterator<N>::type project(IteratorType it);
  template<int N,typename IteratorType>
    typename nth_index_const_iterator<N>::type project(IteratorType it)const;
  template<typename Tag,typename IteratorType>
    typename index_iterator<Tag>::type project(IteratorType it);
  template<typename Tag,typename IteratorType>
    typename index_const_iterator<Tag>::type project(IteratorType it)const;
};

复杂度

关于 multi_index_container 各种操作的描述，请见复杂度说明一节。

实例化类型

multi_index_container 可依以下类型实例化：

  1. Value 为容器中元素的类型，要求为 Assignable可赋值。
  2. IndexSpecifierList 指定组成multi_index_container的索引。它必须是一个非空的 MPL Forward Sequence (最好是 MPL Random Access Sequence)，由索引说明组成。为方便起见，可使用 indexed_by MPL sequence。
  3. Allocator 必须符合 C++ 对分配器的要求 [lib.allocator.requirements]. 允许以下对标准要求的一些放宽：
    • 支持不等价分配器实例：交换两个不等价分配器必须不能抛出任何异常。
    • 对于每个类型 T, 类型 Allocator::rebind<T>::other::pointer 可以是任何一种 随机访问迭代器，它可以从数字0显式构造(相当于空指针)或从任何一个类型为 T* 的 p 构造，指向一个由某个 Allocator 实例其它从 Allocator 重绑定的分配器所分配的区域。对从 p 构造的指针进行提领应得到 *p.
一个 multi_index_container 的索引不能有重复的 tags，不论是在一个索引中还是在两个不同的索引中。

嵌套类型

ctor_args_list
虽然不是严格的定义，但是从用户看来，ctor_args_list 可被视为 ::boost::tuple<C0,...,CI-1>，其中 Ci 为multi_index_container的第i个索引的 ctor_args 类型。严格说来，有一个从 const ::boost::tuple<C0,...,CI-1>& 到 const ctor_args_list& 的隐式类型转换。该类型提供了multi_index_container的索引的构造参数。ctor_args_list 是 Default Constructible可缺省构造的，如果所有 ctor_args 类型都是可缺省构造的。
index_specifier_type_list
相当于 IndexSpecifierList.
index_type_list
以 MPL Random Access Sequence 和 MPL Extensible Sequence 为原型，包含 multi_index_container 所持有的各个索引的类型，与它们声明时的顺序一样。
iterator_type_list
以 MPL Random Access Sequence 和 MPL Extensible Sequence 为原型，包含 multi_index_container 所持有的各个索引的迭代器类型，与它们声明的顺序一样。
const_iterator_type_list
以 MPL Random Access Sequence 和 MPL Extensible Sequence 为原型，包含 multi_index_container 所持有的各个索引的常量迭代器类型，与它们声明的顺序一样。

嵌套类模板

template<int N> struct nth_index
nth_index<N>::type 为 multi_index_container 所持第N个(从0起计)索引的类型，与索引声明的顺序一样。
要求: 0 <= N < I.
template<typename Tag> struct index
index<Tag>::type 返回标志为 Tag 的索引的类型，Tag 为相关的 tag type. 要求: multi_index_container 有以 Tag 为标志的索引。
template<int N> struct nth_index_iterator
nth_index_iterator<N>::type 等同于 nth_index<N>::type::iterator.
template<int N> struct nth_index_const_iterator
nth_index_const_iterator<N>::type 等同于 nth_index<N>::type::const_iterator.
template<typename Tag> struct index_iterator
index_iterator<Tag>::type 等同于 index<Tag>::type::iterator.
template<typename Tag> struct index_const_iterator
index_const_iterator<Tag>::type 等同于 index<Tag>::type::const_iterator.

构造，复制和赋值

explicit multi_index_container(
  const ctor_args_list& comp=ctor_args_list(),
  const allocator_type& al=allocator_type());
效果: 使用给定的参数列表和分配器构造一个空的 multi_index_container.
复杂度: 常量时间.
template<typename InputIterator>
multi_index_container(
  InputIterator first,InputIterator last,
  const ctor_args_list& comp=ctor_args_list(),
  const allocator_type& al=allocator_type());
要求: InputIterator 为一个 Input Iterator输入迭代器，其指向类型 Value 或可转换为 Value 的某个类型。last 可以从 first 到达。效果: 使用给定的参数列表和分配器构造一个空的 multi_index_container，并将区间 [first,last) 的元素填入容器。每个元素的插入操作可能成功也可能不成功，这取决于 multi_index_container 的所有索引是否接受。
复杂度: O(m*H(m)), m 为 [first,last) 中的元素数量。
multi_index_container(
  const multi_index_container<Value,IndexSpecifierList,Allocator>& x);
效果: 构造 x 的一个拷贝，复制它的元素及内部对象(键提取器、比较对象、分配器)。
后续条件: *this==x. multi_index_container 中的每个索引的顺序是被保证的。
复杂度: O(x.size()*log(x.size()) + C(x.size())).
~multi_index_container()
效果: 销毁 multi_index_container 及其包含的所有元素。元素销毁的顺序并不确定。
复杂度: O(n).
multi_index_container<Value,IndexSpecifierList,Allocator>& operator=(
  const multi_index_container<Value,IndexSpecifierList,Allocator>& x);
用 x 的拷贝替换 multi_index_container 中的元素和内部对象。
后续条件: *this==x. multi_index_container 中的每个索引的顺序是被保证的。
返回: *this.
复杂度: O(n + x.size()*log(x.size()) + C(x.size())).
异常安全性: 强，如果 ctor_args_list 中的类型的复制和赋值操作不抛出异常的话。
allocator_type get_allocator()const;
返回 allocator_type 对象的一个拷贝，用于构造 multi_index_container.
复杂度: 常量时间。

投影操作

给定一个带有索引 i1 和 i2 的 multi_index_container，我们说 i1-迭代器 it1 和 i2-迭代器 it2 是相等的，如果：

  • it1==i1.end() 且 it2==i2.end(),
  • 或者 it1 和 it2 指向同一个元素。
template<int N,typename IteratorType>
typename nth_index_iterator<N>::type project(IteratorType it);
要求: 0 <= N < I. IteratorType 属于 iterator_type_list. 它是 *this 某个索引的有效迭代器 (即不是指向其它 multi_index_container的)。
效果: 返回一个与 it 相等的 nth_index_iterator<N>::type 迭代器。
复杂度: 常量时间。
异常安全性: 不抛出异常。

序列化

multi_index_containers 可以借助 Boost.Serialization 进行归档/取回。Boost.MultiIndex 并不提供公有的序列化接口，而是由 Boost.Serialization 提供。普通格式和XML格式的归档都可以支持。

每个索引都含有一个给定的 multi_index_container 作为前提条件，以保证可以取回容器。为了描述这一点，需要使用以下概念。一个类型T被称为可序列化(包括可XML序列化)，如果任何T的对象可以被保存到一个输出的档案(或XML档案)中，并在稍后从同一个档案(或XML档案)中取回。如果 x' 为类型T而且是从对象 x 保存的序列化信息中取出的，则我们说 x' 是 x 的一个恢复拷贝。给定一个 Binary Predicate二元谓词 Pred 作用于 (T, T), 且对象 p 和 q 为类型 Pred, 我们说 q 与 p 是 serialization-compatible序列化兼容的，如果

p(x,y) == q(x',y')
对于每一对类型T的对象 x 和 y 以及 x 和 y 的恢复拷贝 x' 和 y' 都成立。

操作：保存一个 multi_index_container m 到一个输出存档(XML存档) ar.
要求：Value 是可序列化的(XML-可序列化)。另外，m 的每一个索引都可能增加其它要求。
异常安全性：对于 m 是强异常安全的。如果有异常抛出，ar 可能处于不一致状态。
操作：从一个输入存档(XML存档) ar 中导入一个 multi_index_container m'.
要求：Value 是可序列化的(XML-可序列化)。另外，m' 的每一个索引都可能增加其它要求。
异常安全性：基本。如果有异常抛出，ar 可能处于不一致状态。

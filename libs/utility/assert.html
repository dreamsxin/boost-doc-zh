assert.hpp
 

头文件 <boost/assert.hpp> 定义了宏 BOOST_ASSERT, 它与定义于 <cassert> 的标准 assert 宏很相似。这个宏被规定用于 Boost 库。

缺省地，BOOST_ASSERT(expr) 等同于 assert(expr).

当包含 <boost/assert.hpp> 时，如果宏 BOOST_DISABLE_ASSERTS 被定义，则 BOOST_ASSERT(expr) 被定义为 ((void)0). 这可以让用户有选择地关闭 BOOST_ASSERT 而不会影响到标准 assert 的定义。

当包含 <boost/assert.hpp> 时，如果宏 BOOST_ENABLE_ASSERT_HANDLER 被定义，则 BOOST_ASSERT(expr) 对 expr 求值，且如果结果为 false, 则执行表达式

::boost::assertion_failed(#expr, BOOST_CURRENT_FUNCTION, __FILE__, __LINE__)

assertion_failed 声明在 <boost/assert.hpp> 中，如下：

namespace boost
{

void assertion_failed(char const * expr, char const * function, char const * file, long line);

}

但是它没有被定义。这要求用户给出一个合适的定义。

和 <cassert> 一样，<boost/assert.hpp> 可以在一个编译单元中多次包含。BOOST_ASSERT 会象上述那样每次重新定义。

<boost/assert.hpp> 还定义了宏 BOOST_VERIFY. 它具有与 BOOST_ASSERT 一样的行为，除了传递给 BOOST_VERIFY 的表达式一定会被求值。这一点在断言表达式需要具有副作用时是有用的；它也可以在某个变量仅在断言中被使用时，抑制相关的警告。

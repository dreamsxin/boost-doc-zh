占位符

摘要

namespace placeholders {
typedef unspecified _;
typedef arg<1>      _1;
typedef arg<2>      _2;
...
typedef arg<n>      _n;
}

using placeholders::_;
using placeholders::_1;
using placeholders::_2;
...
using placeholders::_n;

描述

占位符的形式为 _n，它只是 arg<n> 特化物的同义词。无数字的占位符 _ (下划线)在绑定和 lambda 表达式中具有 特殊的意义，而且在这些上下文以外不具有已定义的语义。

通过 using namespace mpl::placeholders; 指令，占位符的名字可以在用户的名字空间中使用。

头文件

#include <boost/mpl/placeholders.hpp>

[注: 在使用占位符构造一个 Lambda 表达式 用于传递给MPL的算法或元函数时，可以省略这个 include：本库中所有接受 lambda 表达式的组件都可以让占位符在用户的代码中直接可用 — 注完]

参数

无。

表达式语义

对于区间 [1, BOOST_MPL_LIMIT_METAFUNCTION_ARITY] 中的任意整型常量 n 以及任意类型 a1,... an:

typedef apply_wrapn<_n,a1,...an>::type x;
返回类型: 一个类型
语义:

等价于

typedef apply_wrapn< arg<n>,a1,...an >::type x;

例子

typedef apply_wrap5< _1,bool,char,short,int,long >::type t1;
typedef apply_wrap5< _3,bool,char,short,int,long >::type t3;

BOOST_MPL_ASSERT(( is_same< t1, bool > ));
BOOST_MPL_ASSERT(( is_same< t3, short > ));

参见

合成与参数绑定, arg, lambda, bind, apply, apply_wrap

描述

给定一个提供对单个字符序列访问的过滤器或设备，我们可以构造另一个过滤器或设备，提供对原序列中的一段连续的子序列的访问。第二个过滤器或设备被称为原设备的约束。约束以用类模板 restriction 的实例来表示。函数模板 resrict 是一个 对象生成器，在传入一个过滤器或设备以及一对用于表示约束子序列的端点的值时，它返回一个合适的 resriction 实例。

一个 resriction 的 mode 与其底层的组件相同。resriction 是 Closable, Flushable, Localizable 和 OptimallyBuffered 的。

函数模板 slice 是 restrict 的一个别名，用于将 restrict 作为关键字的平台。

概要

namespace boost { namespace iostreams {

template<typename Component>
class restriction {
public:
    typedef typename char_type_of<Component>::type  char_type;
    typedef implementation-defined                  mode;

    restriction( [const] Component& component,
                 stream_offset off, 
                 stream_offset len = -1 );

    // 过滤器或设备的成员函数
};

template<typename Component>
restriction<Component> 
restrict( [const] Component& component,
          stream_offset off, 
          stream_offset len = -1 );

} } // End namespace boost::io

类模板 restriction

模板参数

Component - 一个 Filter 或 Device 的 model

restriction::restriction

    restriction( [const] Component& component,
                 stream_offset off, 
                 stream_offset len = -1 );

基于给定的组件和偏移量构造一个 restriction 实例。其中的参数解释如下：

component - 被约束的过滤器或设备。如果 Component 是一个流或流缓冲类型，则函数参数是一个非-const 引用；否则是一个 const 引用
off - 约束子序列的开始位置偏移量
len - 约束子序列的长度。值 -1 表示约束子序列的末尾与原序列末尾相同

Flushable可刷新的

描述

可刷新的接收器或输出过滤器提供了一个手段，用户可以请求将所有缓冲的字符发送到下游。

强化自

关联类型

与 Filter 或 Device 相同，有以下额外要求：

Category 一个类型，可转换为 flushable_tag

符号

F - 一个类型，是 Flushable 的 model
D - 一个类型，是 Sink 的 model，具有与 F 相同的字符类型
f - 类型 F 的对象
d - 类型 D 的对象

有效表达式 / 语义

表达式 表达式类型 前提条件 语义 
boost::iostreams::flush(f)
 bool Category 可转换为 device_tag 尝试将所有缓冲的字符发送到下游，返回 true 除非有错误发生。[1] 
boost::iostreams::flush(f, d)
 bool Category 可转换为 filter_tag

尝试写出所有缓冲字符到 d，返回 true 如果所有缓冲字符全部写出

异常

在 flush 的执行过程中发生的错误将通过抛出异常来表示。如果这些异常是在流或流缓冲的析构函数中抛出的话，则捕获并忽略它们。

模型

Iostreams 库提供的几个过滤器和设备都是可刷新的，但是这属于实现细节。

[1]在后续的开发中，为了和以异常方式报告错误的策略(见 Exceptions)相一致，当 flush 用于设备时将会返回 void. 在规范被修改之前，设备在刷新时总是返回 true.

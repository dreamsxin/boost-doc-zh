Peekable可查看的

描述

可查看的 Device 允许从输入序列中读取一个字符后又退回去。

这个概念的名字来自于对 PeekableDevice 的观察，该设备允许用户查看其控制的序列中的第一个字符，在调用 get 或 read 后紧跟一个 putback. 名字 Putbackable 和 InputRevertable 也被考虑过采用但最后还是拒绝了。

强化自

Source.

关联类型

与 Source 一致。

符号

P - 一个类型，为 Peekable 的 model
p - 类型 P 的对象
c - 与 P 的字符类型相同的对象
io - 名字空间 boost::iostreams 的别名

有效表达式 / 语义

表达式 表达式类型 前提条件 语义 后续条件 
io::putback(p, c)
 bool 已使用 io::get 或 io::read 从 p 中读取一个或多个字符，且未执行其它 i/o 操作 尝试将 c 放回 p，返回 true 表示成功 如果 putback 返回 true, 则使用 io::get 或 io::read 从 p 读取的下一个字符将等于 c

异常

在 putback 的执行过程中发生的错误将以抛出异常来表示。

模型

  • 标准输入流和流缓冲。

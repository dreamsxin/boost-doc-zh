SeekableFilter可定位过滤器

定义

SeekableFilter 是一个 Filter，其 mode 强化自 seekable.

描述

可定位过滤器操作于由 SeekableDevice 控制的字符序列之上，提供对具有相同字符类型的被过滤序列的访问。它有两种方法处理被过滤序列：

  1. 定义成员函数 get, put 和 seek.
  2. 定义成员函数 read, write 和 seek.
第二种方法用于提高性能。实现了该方法的可定位过滤器被称为 多字符的. (见 Multi-Character Filter.)

强化自

InputFilter, OutputFilter.

关联类型

Character type 被过滤序列中的字符类型
Category 一个类型，可转换为 filter_tag 和 seekable
Mode Category 可转换为的，唯一的，位于派生最深层的 mode tag

 可转换为 multi_char_tag 从由 f 控制的输入序列读入最多 n 个字符到缓冲区 s1, 返回读入的字符数。返回小于 n 的值表示序列结束。可以通过 io::read 和 io::putback 访问 d 所控制的输入序列。

std::streamsize

f.seek(d, off, way)
 std::streampos -

将读/写指针前进 off 个字符，返回新的位置，其中偏移的计算从以下位置开始：

  • 如果 way 为 ios_base::beg 则从序列始端计
  • 如果 way 为 ios_base::cur 则从当前位置计
  • 如果 way 为 ios_base::end 则从序列末端计

可以通过 io::seek 访问 d 所控制的输入序列，如果 F 的 mode 可转换为 output 则使用 io::write .

异常

在 get, put, read, write 或 seek 的执行过程中发生的错误将通过抛出异常来表示。到达输入序列的末尾并不是错误，但试图向超过输出序列末尾的地方写入则是错误。

抛出异常后，可定位过滤器必须处于一致的状态；后续的 i/o 操作可以抛出异常但必须具有良好定义的行为。此外，除非过滤器是 Closable, 否则它必须准备好处理新的字符序列。

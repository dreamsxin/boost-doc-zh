核心组件

以下组件形成了 Boost.Iostreams 和标准 iostreams 库间的接口。

名字 头文件[1] 描述
stream stream.hpp Stream 模板使用其第一个模板参数的实例来执行 i/o, 该参数必须符合概念 Device. 根据其第一个模板参数是否符合 来源, 接收器 或两者，决定它派生自 std::basic_istream, std::basic_ostream 或 std::basic_iostream .
stream_buffer stream_buffer.hpp Stream buffer 模板使用其第一个模板参数的实例来执行 i/o, 该参数必须符合概念 Device.
filtering_stream filtering_stream.hpp Stream 模板使用内含的 chain 执行过滤 i/o. 根据其 mode ，决定它派生自 std::basic_istream, std::basic_ostream 或 std::basic_iostream, 模式由第一个模板参数指定
filtering_streambuf filtering_streambuf.hpp Stream buffer 模板使用内含的 chain 执行过滤 i/o.
chain chain.hpp 零个或多个 Filters 组成的序列，后跟一个可选的 Device, 以类似于栈的接口访问。由 filtering_stream 和 filtering_streambuf 使用。
code_converter code_converter.hpp 设备适配器，带一个窄字符 Device，通过引入一个 code conversion 层生成一个宽字符 Device.

算法

算法 头文件 描述
copy boost/iostreams/copy.hpp

从一个 Source 读入字符，写出到一个 Sink，直到流结束；返回写出的字符数

视图

每个视图包含一个或多个类模板，由一个 object generator 组合在一起。见 Views.

对象生成器 视图模板 头文件[1] 描述
combine combination combine.hpp 带一对 Filters 或 Devices，生成一个过滤器或设备，用第一个组件执行输入，用第二个组件执行输出
compose composite compose.hpp 带一个 Filter 以及一个 Filter 或 Device，生成一个过滤器或设备，输出流按顺序通过两个组件，而输入流则以相反顺序通过两个组件
invert inverse invert.hpp 带一个 Filter，生成一个过滤器，执行相同的过滤算法，如果给定的组件是 InputFilter 则转为 OutputFilter，反之亦然
restrict restriction restrict.hpp 带一个 Filter 或 Device 及一个流偏移量和一个可选的长度，生成一个 Filter 或 Device，访问给定组件中的指定子序列
tee tee_device tee.hpp 带一对 Sinks，生成一个接收器，将所有流入的数据都转发到两个给定接收器
tee_filter 带一个 Filter 和一个 Sink，生成一个过滤器，将所有流出的数据转发到接收器并同时转到下游

注释

[1]路径名相对于目录 boost/iostreams.

[2]路径名相对于目录 boost/iostreams/device.

[3]路径名相对于目录 boost/iostreams/filter.

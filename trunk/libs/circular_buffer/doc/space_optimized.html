Description

circular_buffer_space_optimized 容器是 circular_buffer 的一个适配器。circular_buffer_space_optimized 的功能类似于基类 circular_buffer，只是它不是在创建时一次性分配内存，而是在需要时才分配内存。(预先的内存分配类似于典型的 std::vector 实现)。此外，当容器缩小时会自动释放内存。

图： 空间优化循环缓冲区的内存分配过程。capacity controller 的 min_capacity() 表示被分配内存的最小保证数量。被分配内存不会低于该值。min_capacity() 的缺省值设置为 0.

注意，如果功能相同，则相关链接会指向原本的 circular_buffer.

原理

空间优化循环缓冲区的自动调整大小模式可用于容器可能保存大量元素而多数时候只保存少量元素的情况。使用 circular_buffer_space_optimized 可以减少内存的消耗并提高CPU cache的效率。

头文件

circular_buffer_space_optimized 定义于文件 boost/circular_buffer.hpp. 在头文件 boost/circular_buffer_fwd.hpp 中也有 circular_buffer_space_optimized 的前向声明。

模型概念

随机访问容器Random Access Container, 前端插入序列Front Insertion Sequence 和 后端插入序列Back Insertion Sequence.

特定公有类型

以下公有类型是 circular_buffer_space_optimized 特有的。与 circular_buffer 相同的公有类型请见 circular_buffer 的文档。

类型 说明
capacity_type 空间优化循环缓冲区的容量控制器。 
class capacity_control {
   size_type m_capacity;
   size_type m_min_capacity;
public:
   capacity_control(size_type capacity, size_type min_capacity = 0) : m_capacity(capacity), m_min_capacity(min_capacity) {};
   size_type capacity() const { return m_capacity; }
   size_type min_capacity() const { return m_min_capacity; }
   operator size_type() const { return m_capacity; }
};
前提条件：
capacity >= min_capacity
capacity() 表示 circular_buffer_space_optimized 的容量，而 min_capacity() 决定其内部缓冲区的最小分配大小。capacity_control 的转换构造函数可以从类似于 size_type 的类型隐式转换，以确保可以用其它STL容器创建一个 circular_buffer_space_optimized 实例的兼容性。另一方面，操作符 size_type() 提供了到 size_type 的隐式转换，这样可以用和 circular_buffer 一样的方法处理 circular_buffer_space_optimized 的容量。

template <class InputIterator>
circular_buffer_space_optimized(InputIterator first, InputIterator last, const allocator_type& alloc = allocator_type());

以指定区间的拷贝创建一个满的空间优化循环缓冲区。
前提条件：
有效区间 [first, last).
first 和 last 必须符合 输入迭代器InputIterator 的要求。
作用:
capacity().capacity() == std::distance(first, last) && capacity().min_capacity() == 0 && full() && (*this)[0]== *first && (*this)[1] == *(first + 1) && ... && (*this)[std::distance(first, last) - 1] == *(last - 1)

内部缓冲区的分配内存容量为 std::distance(first, last).
参数:
first
被复制区间的开始。
last
被复制区间的结束。
alloc
分配器。
抛出:
如果内存耗尽则抛出内存分配错误(std::bad_alloc 如果使用标准分配器)。
T::T(const T&) 所抛出的任何异常。
复杂度:
线性(与 std::distance(first, last) 相关)。
template <class InputIterator>
circular_buffer_space_optimized(capacity_type capacity_ctrl, InputIterator first, InputIterator last, const allocator_type& alloc = allocator_type());

以指定容量(及分配内存的最小保证容量)创建一个空间优化循环缓冲区，填充以指定区间的拷贝。
前提条件：
有效区间 [first, last).
first 和 last 必须符合 输入迭代器InputIterator 的要求。
作用:
capacity() == capacity_ctrl && size() <= std::distance(first, last) && (*this)[0]== (last - capacity_ctrl.capacity()) && (*this)[1] == *(last - capacity_ctrl.capacity() + 1) && ... && (*this)[capacity_ctrl.capacity() - 1] == *(last - 1)

如果从区间 [first, last) 复制的元素数量大于给定的 capacity_ctrl.capacity()，则只有区间 [last - capacity_ctrl.capacity(), last) 中的元素被复制。

内部缓冲区的分配内存容量为 max[capacity_ctrl.min_capacity(), min[capacity_ctrl.capacity(), std::distance(first, last)]].
参数:
capacity_ctrl
容量控制器，表示可以存放在 circular_buffer_space_optimized 中的最大元素数量和内部缓冲区的最小分配大小。
first
被复制区间的开始。
last
被复制区间的结束。
alloc
分配器。
抛出:
如果内存耗尽则抛出内存分配错误(std::bad_alloc 如果使用标准分配器)。
T::T(const T&) 所抛出的任何异常。
复杂度:
线性(与 std::distance(first, last) 相关；如果 InputIterator 为 随机访问迭代器RandomAccessIterator 则与 min[capacity_ctrl.capacity(), std::distance(first, last)] 相关)。

修改 circular_buffer_space_optimized 的容量(及分配内存的最小保证容量)。
作用:
capacity() == capacity_ctrl && size() <= capacity_ctrl.capacity()

如果当前保存在 circular_buffer_space_optimized 中的元素数量大于所希望的新容量，则最后 [size() - capacity_ctrl.capacity()] 个元素被删除，新的大小等于 capacity_ctrl.capacity().

如果当前保存在 circular_buffer_space_optimized 中的元素数量小于新的容量，则内部缓冲区中的分配内存容量可能被按需要调整，但不会低于 capacity_ctrl.min_capacity().

capacity_ctrl

内部缓冲区的已分配内存数量可能预先减少。
参数:
first
被删除区间的开始。
last
被删除区间的结束。
返回:
指向被删除元素之后第一个元素的迭代器，如果不存在这样的元素则返回 end().
抛出:
如果内存耗尽则抛出内存分配错误(std::bad_alloc 如果使用标准分配器)。
T::operator = (const T&) 抛出的任何异常。
异常安全性:
基本。
迭代器失效:
指向 circular_buffer_space_optimized 的所有迭代器(除了等于 end() 的迭代器)均失效。
复杂度:
线性(与 circular_buffer_space_optimized 的大小相关)。
参见:
erase(iterator), rerase(iterator), rerase(iterator, iterator), clear()
iterator rerase(iterator pos);

删除指定位置的一个元素。 
前提条件：
pos 是一个指向 circular_buffer_space_optimized 的有效迭代器(但不是 end())。

内部缓冲区的已分配内存数量可能预先减少。
作用:
位置 pos 的元素被删除。
参数:
pos
指向被删除元素的迭代器。
返回:
指向被删除元素之前第一个元素的迭代器，如果不存在这样的元素则返回 begin().
抛出:
如果内存耗尽则抛出内存分配错误(std::bad_alloc 如果使用标准分配器)。
T::operator = (const T&) 抛出的任何异常。
异常安全性:
基本。
迭代器失效:
指向 circular_buffer_space_optimized 的所有迭代器(除了等于 end() 的迭代器)均失效。
复杂度:
线性(与 circular_buffer_space_optimized 的大小相关)。
说明：
基本上这个方法与 erase(iterator) 没有不同。它只是为了保持与基类 circular_buffer 的一致性而实现。
参见:
erase(iterator), erase(iterator, iterator), rerase(iterator, iterator), clear()

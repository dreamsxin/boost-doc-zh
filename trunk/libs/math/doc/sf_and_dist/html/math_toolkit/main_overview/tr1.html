C99 和 TR1 中的 C-风格 函数概览

这个库中所包含的大量的特殊函数是 C99 标准 ISO/IEC 9899:1999 或 C++技术报告之库扩展的一部分。因此这个库包含了一个小的函数包装头文件boost/math/tr1.hpp 用于提供与这两个标准的兼容性.

在这种方式下使用这个库有各种 pros 和 cons:

优点：

  • 包含的头文件是轻量级的 (也就是. 快速编译).
  • 这些函数为 extern "C" 链接, 因此是从其它语言(不仅是C和C++)中引入的.
  • C99 和 C++ TR1 标准兼容性.

缺点：

  • 你需要编译和链接其它的外部 Boost.Math 库.
  • 受限的支持类型, float, double 和 long double.
  • 错误处理通过设置 ::errno 以及返回 NaN's 和 无限数: 与C++基于异常处理的方式相比 ，这种方式不那么灵活.

注意

仅 当你选择使用 boost/math/tr1.hpp 而不是其它的Boost.Math 头文件时才需要其它的库，其它的 Boost.Math仅保留头文件.

为了使用 tr1.hpp 而需要的其它Boost分离库可以使用 bjam 在 libs/math/build 目录下进行编译, 或者在 Boost 根目录下使用 Boost-wide 安装过程. 在 libs/math/src 中的头文件且它们实现的的函数具有相同的名字. 各种库的命名如下:

其中 <suffix> 编码编译器以及用于编译这个库的编译选项: 例如"libboost_math_tr1-vc80-mt-gd.lib"是用于Visual C++ 8.0的静态链接多线程调试模式的TR1 库, 使用 DLL VC++ 运行时支持, 而 "boost_math_tr1-vc80-mt.lib" 是使用Visual C++ 8.0发行多线程编译模式运行时的TR1DLL导入库. 参看使用指导<suffix> 含义完整解释.

注意

基于你的编译设置，Visual C++通常会有正确的编译链接选项而不是由 boost/math/tr1.hpp来选择的编译选项.

当编译代码的时候， 如果用户想要链接这些库的 DLL 版本而不是静态版本. 用户需要定义 BOOST_MATH_TR1_DYN_LINK。

用户可以通过定义 BOOST_MATH_TR1_NO_LIB 来禁止编译时的自动链接: 这仅当链接一个用户编译的库时使用.

注意

Linux 和 Unix 用户通常只有这个已安装库的一个链接变量 且通常链接 -lboost_math_tr1 等等.

使用建议

这些库现在给用户一个选择:

  • 包含这些函数仅有头文件的版本，这就会有一个短的链接时间, 但是会有一个长的编译时间.
  • 包含 TR1 头文件同时链接一个外部库.

选择哪一各方式很大程度上依赖于你选择如何工作以及你的系统是如何安排的.

例如一个临时的用户仅仅需要使用acosh 函数, 或许最好包含头文件<boost/math/special_functions/acosh.hpp> 然后在他的代码中使用用boost::math::acosh(x).

然而, 对于编译时间很长的大规模软件开发, 且 Boost 库已经编译安装在系统中, 那么包含 <boost/math/tr1.hpp>并使用boost::math::tr1::acosh(x) 将会加快编译速度, 减少对象(obj)文件大小 (因为不再有模板实例化), 同样也会加快调试运行时间 - 因为外部编译的库可以被编译器优化 , 而不是使用完整设置来编译 - 发行版 和 调试版的编译时间相差20倍, 因此对于大的项目这是一个巨大的收益.

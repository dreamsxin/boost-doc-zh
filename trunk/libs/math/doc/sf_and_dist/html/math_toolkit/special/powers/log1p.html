log1p 

#include <boost/math/special_functions/log1p.hpp>


namespace boost{ namespace math{

template <class T>
calculated-result-type log1p(T x);

template <class T, class Policy>
calculated-result-type log1p(T x, const Policy&);

}} // namespaces

返回x+1的自然对数。

函数返回值的类型使用返回值推导法则:来确定，当x是整型的时候，返回值的类型是double,否则返回值是类型是T.

最后一个策略 参数是可选的并且可以用来控制函数的行为: 如何处理错误, 使用哪种层次的精度等等. 参见策略文档了解更多信息。

有许多计算log(x+1)的情况是可取的， 然而,对于较小的x值 那么x+1 将会 受到巨量消失误差(catastrophic cancellation errors)以至于x+1 == 1 且log(x+1) == 0, 而事实上对于较小的 x, 对于log(x+1) 最佳逼近值是 x. log1p 函数出于精度的考虑使用泰勒级数展开来计算log(1+x)最佳逼近值 (小于2个epsilon). 另一方面，请注意:存在更快的计算方法, 使用下面的等价关系: 

log(1+x) == (log(1+x) * x) / ((1-x) - 1)

然而, 经验显示，当打开编译器的优化选项的时候，这些函数将会导致很明显的失败，因此只会在一个已知的编译器中上面的方法不会失效时才会使用这个方法. 相反, 级数展开的方法对于编译器优化导致的错误似乎具有免疫.

最后，当定义了 BOOST_HAS_EXPM1 时，float/double/long double 这些模板的特化只是简单的调用这些平台(POSIX)定义的函数。

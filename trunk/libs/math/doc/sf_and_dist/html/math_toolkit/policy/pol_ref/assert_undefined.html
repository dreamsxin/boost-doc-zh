数学上未定义的函数策略(Mathematically Undefined Function Policies)

有一些函数很通用 (对于所有支持的分布类型都提供了)但是对于一些特定的分布可能在数学上没有定义这些通用的函数。

例如，柯西分布( Cauchy distribution )就没有均值，因此下面的代码应当返回什么值呢？

mean(cauchy<>());

这种代码是否都不能通过编译呢？

缺省的行为是这些函数不能编译通过－实际上它们将会产生一个静态断言(static assertion) －但是通过改变策略我们可以让这些函数返回一个定义域错误(依赖于错误处理策略，也可能抛出一个异常)。

这种行为由assert_undefined<> 策略控制：

namespace boost{ namespace math{ namespace policies {

template <bool b>
class assert_undefined;

}}} //namespaces

例如：

#include <boost/math/distributions/cauchy.hpp>

using namespace boost::math::policies;
using namespace boost::math;

// 这种代码不能通过编译，柯西分布没有均值!
double m1 = mean(cauchy()); 

// 这可以通过编译，但是会产生一个定义域错误!
double m2 = mean(cauchy_distribution<double, policy<assert_undefined<false> > >());

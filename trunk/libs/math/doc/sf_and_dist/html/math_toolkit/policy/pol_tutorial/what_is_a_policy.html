什么是策略(So Just What is a Policy Anyway)?

策略是一个编译时的机制，用来定义一个特殊函数的行为，或者一个统计分布的行为。使用策略你可以控制：

  • 当遇到错误时如何处理。
  • 当你调用一个数学上未定义的函数时，会发生什么(例如你计算一个柯西分布的均值)。
  • 当你计算一个离散分布的分位点时，会发生什么。
  • 为了改进精度，在这个库的内部是否能够进行从float到double，double到long double的类型提升。
  • 使用哪种精度计算结果。

这些策略中的一些可以是运行时变量，但是我们就不能在编译时为这些策略指定最好的计算方法。

出于这个理由 策略是一种类型 ：实际上，它是boost::math::policies::policy<>类模板的一个实例(instance)。这个类是一个用户选定的策略的编译时容器(compile-time-container) 。 

using namespace boost::math::policies;
//
// 定义一个在出现溢出错误时设置 ::errno 
// 并在内部不进行从 double 到 long double 的类型提升:
//
typedef policy<domain_error<errno_on_error>, promote_double<false> > mypolicy;

Quaternion 创建函数 

template<typename T>	quaternion<T>	spherical(T const & rho, T const & theta, T const & phi1, T const & phi2);
template<typename T>	quaternion<T>	semipolar(T const & rho, T const & alpha, T const & theta1, T const & theta2);
template<typename T>	quaternion<T>	multipolar(T const & rho1, T const & theta1, T const & rho2, T const & theta2);
template<typename T>	quaternion<T>	cylindrospherical(T const & t, T const & radius, T const & longitude, T const & latitude);
template<typename T>	quaternion<T>	cylindrical(T const & r, T const & angle, T const & h1, T const & h2);

这些函数通过类似于极坐标构造复数的方式来构造四元数, 因为没有严格等价的相对应于四元数的极坐标系.

球面 是一种简单的 极坐标变换, 它以一个距离(magnitude)（正的），以及一个超球面上的由三个角度给定的点作为输入 第一个角度, theta 有一个固有的 从-pi 到 +pi 的范围, 而其它的两个角度有一个固有的从 -pi/2 到 +pi/2的范围 (类似于R3 空间上的球面坐标系). 由于对称性和周期性, 由于存在大量的对称性和周期性，如果这个距离是负数 ( 译注：严格意义上来讲，距离是非负的，作者这里所说的 距离为负数是指使用这个函数的程序员将一个负值作为距离值传递给这个函数) 或者这个角度超过 了它的自然的分布范围也不会有什么麻烦的问题产生. 然而，预期的退化仍然会发生(一个大小为0的距离将会使得角度设置值被忽略...).

圆柱 也是柱面坐标系在通常的R3 空间(译注：就是数学中通常所说的三维坐标空间)上的变换, 柱面坐标反过来也是平面极坐标的另一个派生. 最开始的两个输入(函数的最开始的两个参数)是这个四元数的第一个 C ( 译注：:在数学中 C 表示复数域) 成员的极坐标. 第三个和第四个输入（函数的第三个和第四个参数）分别放入这个四元数的R ( 译注：:在数学中 R 表示实数域) 成员中.

多极（multipolar） 是极坐标系的另一个泛化. 这一次, 四元数的两个C 成员都由极坐标给定.

cylindrospherical 特定于四元数. 通常把H(四元数集) 看作是R 与 R3的笛卡尔积是一件 有趣的事情(四元数乘法因此有一种特殊的形式, 就像这里给定的一样). 这个函数因此基于这种表现形式来构造一个四元数, R3 成员由通常的R3 球面坐标给定.

半极化(semipolar) 是另一个特定于 四元数的发生器(generator).它把第一个输入参数作为四元数的大小, 第二个参数是一个范围在 0 到 +pi/2 的角度，进而 这个四元数的最开始的两个 C 成员的大小 分别是第一个参数和这个角度的sin 和 cos 乘积, 最后的第三个和第四个参数是范围在-pi/2 到 +pi/2 的角度，分别用于描述这个四元数的第一个和第二个C components 成员的大小变化. 像平常一样, 如果这个距离是负数或者这些角度超过 了它的自然的分布范围也不会有什么麻烦的问题产生, 因为对称性和周期性很好的解决了这些问题.

在这种实现版本的四元数中, 并没有以复数值为参数的构造四元数的操作，因为这种情况颇为复杂. 四元数可以与R3 和 R4 中的旋转相关联起来, 并且这种联系不是太复杂, 但是这一点仍然缺少标准的矩阵库(事实上或理论上的) 能进行这种转换. 这一点可能会在将来的版本中进行改进. 同时, 这里提供了一个如何进行这种转换的针对于 R3, 和针对于 R4 的例子 (例子测试文件).

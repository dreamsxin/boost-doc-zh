EventVisitor Concept 事件遍历器概念

这个概念定义了单事件遍历器的接口。事件遍历器(EventVisitor)具有一个应用成员函数(operator())，它在图算法的内部被调用，调用的时间点是由事件遍历器内的 event_filter typedef 所指定的事件点。多个事件遍历器可以合并到一个 事件遍历器列表EventVistorList 中。

以下是可以在BGL算法中调用的事件标签列表。每个标签对应于用于某个算法的遍历器的一个成员函数。例如，breadth_first_search() 的 BFSVisitor 有一个 cycle_edge() 成员函数。对应的标签是 on_cycle_edge。事件遍历器的 operator() 的第一个参数必须是边描述符或顶点描述符，取决于事件标签。 

namespace boost {
  struct on_initialize_vertex { };
  struct on_start_vertex { };
  struct on_discover_vertex { };
  struct on_examine_edge { };
  struct on_tree_edge { };
  struct on_cycle_edge { };
  struct on_finish_vertex { };
  struct on_forward_or_cross_edge { };
  struct on_back_edge { };
  struct on_edge_relaxed { };
  struct on_edge_not_relaxed { };
  struct on_edge_minimized { };
  struct on_edge_not_minimized { };
} // namespace boost

Refinement of 精化自

可复制构造 Copy Constructible (遍历器的复制应该是一个轻量级的操作)。 

Notation 记号

G 一个以 Graph 为模的类型。
g 一个类型为 G 的对象。
V 一个以 EventVisitor 为模的类型。
vis 一个类型为 V 的对象。
x 一个类型为 boost::graph_traits<G>::vertex_descriptor 或 boost::graph_traits<G>::edge_descriptor 的对象。

Associated Types 关联类型

Event Filter V::event_filter 一个指定遍历器应在哪个事件上调用的标签结构。

Valid Expressions 有效表达式

NameExpressionReturn TypeDescription
遍历器应用 vis(x, g) void 对对象 x 调用遍历器操作，x 是图的顶点描述符或边描述符。

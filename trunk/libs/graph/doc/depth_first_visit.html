depth_first_visit

void depth_first_visit(IncidenceGraph& g,
  typename graph_traits<IncidenceGraph>::vertex_descriptor s, 
  DFSVisitor& vis, ColorMap color)

template <class IncidenceGraph, class DFSVisitor, class ColorMap, 
          class TerminatorFunc>
void depth_first_visit(IncidenceGraph& g,
  typename graph_traits<IncidenceGraph>::vertex_descriptor s, 
  DFSVisitor& vis, ColorMap color, TerminatorFunc func = TerminatorFunc())

该函数使用 深度优先模式 访问与源顶点 s 位于同一个连通分支的所有顶点。该函数的主要目的是为了实现 depth_first_search()，虽然有时候它本身也会被单独使用。

由用户提供的 DFSVisitor 决定了在算法内部的各个事件点上执行什么动作。

ColorMap 由算法用于跟踪哪些顶点已被访问。

也可以用第二种变体，例如，从一个开始顶点找出所有可到达的已标记顶点，从开始顶点到这些顶点有一条不包含任何其它已标记顶点的路径。

Where Defined: 定义于

boost/graph/depth_first_search.hpp

Parameters 参数

IN IncidenceGraph& g
一个有向图或无向图。图类型必须符合 关联图Incidence Graph.
Python: 该参数被命名为 graph. 
IN: vertex_descriptor s
开始搜索的源顶点。
Python: 该参数被命名为 root_vertex. 
IN: DFSVisitor visitor
一个遍历器对象，在算法内部某些由 DFS 遍历器 概念所指定的事件点被调用。这个遍历器对象是以值方式传递的[1]。
Python: 该参数应为派生自该图的 DFSVisitor 类型的一个对象。
UTIL: ColorMap color
被该算法用于跟踪对图的处理过程。类型 ColorMap 必须符合 读/写属性映射Read/Write Property Map 并且其键类型必须为图的顶点描述符类型，值类型必须符合 ColorValue。
Python: 这个颜色映射必须是图的一个 vertex_color_map。 
IN: TerminatorFunc func
一个带两个参数 - 顶点描述符和图实例 - 并返回 bool 的可调用函数对象。该函数对象在调用 'discover_vertex' 遍历器事件后立即被调用。如果返回 true，则该顶点的出边不被检查，就象它们不存在一样。
Python: 不支持该参数。

Complexity 复杂度

时间复杂度为 O(E).

Notes 备注

[1] 由于 visitor 参数是以值方式传递的，所以如果你的遍历器含有状态，则在算法中对该状态的所有修改都是针对该遍历器对象的一个拷贝，而不对传入的遍历器对象进行的。因此你应该让该遍历器以指针或引用的方式保存该状态。

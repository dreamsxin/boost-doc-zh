Blocking阻塞

描述

一个 Device 是 Blocking 的，如果 read 请求从不产生少于请求数量的字符，除了到达流末尾，并且如果 write 请求在任何情况下从不消耗少于请求数量的字符。Iostreams 库当前提供的所有设备都是 Blocking 的。对于非阻塞设备的有限支持已经可以提供，作为未来全面支持 asynchronous and non-blocking i/o 的基础。

Blocking 概念不适用于过滤器。作为替代，过滤器被要求为 blocking-preserving, 意思是

  • read 请求从不产生少于请求数量的字符，除非到达流末尾，或是向下游来源的 read 请求产生了少于请求数量的字符；并且
  • write 请求从不消耗少于请求数量的字符，除非向下游来接收器的 write 请求消耗了少于请求数量的字符。

与 Iostreams 库中的其它概念不同，Blocking 不与任何 category tag 相关联。以后可能会引入 non_blocking_tag .

强化自

Device.

关联类型

与 Device 一样。

有效表达式 / 语义

有效表达式与 Device 相同。额外的语义要求如前所述。

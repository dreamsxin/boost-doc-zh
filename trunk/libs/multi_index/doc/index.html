Boost多索引容器库


Tutorial指南

Boost多索引容器库提供了名为multi_index_container的类模板，可以用于建造拥有一个或多个索引(indices) 的容器，不同的索引具有不同的排序和访问语义。这些索引都提供了类似于STL容器的接口，因此使用起来也非常相似。在一组元素之上维护多个索引的想法来自 于关系数据库，并且考虑到简单的set和map无法满足多索引表中的复杂数据结构的规范。有多种的索引类型可供选用，它们分别模仿了多种STL容器，如 std::set, std::list 和 hashed set。

Boost.MultiIndex 还具有其它功能，如子对象查找、范围查询和就地更新元素，这些功能使得即使在不需要多索引的时候，也值得用它来替换 std::set 和 set::multiset。

Boost.MultiIndex 的多功能还考虑了多种不同的数据结构的特点。以下都是本文档中将使用的例子：

  • 具有多个迭代顺序和查找标准的Sets.
  • 可快速查找及/或无重复的Lists
  • 双向maps, 即可用key或value来查找的maps.
  • MRU (最近使用) lists, 保持最近n个使用的项，从最新一个开始
  • 替代标准容器 可使用Boost.MultiIndex提供的额外功能

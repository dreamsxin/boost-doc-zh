regex_iterator

The iterator type regex_iterator will enumerate all of the regular expression matches found in some sequence: dereferencing a regex_iterator yields a reference to a match_results object.
迭代器类型 regex_iterator 遍历在序列中找到的所有正则表达式匹配：解引用 regex_iterator 会返回指向 match_results 对象的引用。 

template <class BidirectionalIterator, 
         class charT = iterator_traits<BidirectionalIterator>::value_type,
         class traits = regex_traits<charT> >
class regex_iterator 
{
public:
   typedef          basic_regex<charT, traits>                              regex_type;
   typedef          match_results<BidirectionalIterator>                    value_type;
   typedef typename iterator_traits<BidirectionalIterator>::difference_type difference_type;
   typedef          const value_type*                                       pointer;
   typedef          const value_type&                                       reference;
   typedef          std::forward_iterator_tag                               iterator_category;
   
   regex_iterator();
   regex_iterator(BidirectionalIterator a, BidirectionalIterator b, 
                  const regex_type& re, 
                  match_flag_type m = match_default);
   regex_iterator(const regex_iterator&);
   regex_iterator& operator=(const regex_iterator&);
   bool operator==(const regex_iterator&)const;
   bool operator!=(const regex_iterator&)const;
   const value_type& operator*()const;
   const value_type* operator->()const;
   regex_iterator& operator++();
   regex_iterator operator++(int);
};

typedef regex_iterator<const char*>                  cregex_iterator; 
typedef regex_iterator<std::string::const_iterator>  sregex_iterator; 

#ifndef  BOOST_NO_WREGEX
typedef regex_iterator<const wchar_t*>               wcregex_iterator; 
typedef regex_iterator<std::wstring::const_iterator> wsregex_iterator; 
#endif

   regex_iterator<typename std::basic_string<charT, ST, SA>::const_iterator, charT, traits> 
      make_regex_iterator(const std::basic_string<charT, ST, SA>& p,

Description 说明

A regex_iterator is constructed from a pair of iterators, and enumerates all occurrences of a regular expression within that iterator range.
regex_iterator 从一对迭代器构造，遍历迭代器范围内正则表达式所有的出现。 

regex_iterator();

Effects: constructs an end of sequence regex_iterator.
效果：构造一个序列 regex_iterator 的结尾迭代器。 

regex_iterator(BidirectionalIterator a, BidirectionalIterator b, 
               const regex_type& re, 
               match_flag_type m = match_default);

Effects: constructs a regex_iterator that will enumerate all occurrences of the expression re, within the sequence [a,b), and found using match_flag_type m. The object re must exist for the lifetime of the regex_iterator.
效果：构造一个 regex_iterator，用于遍历表达式 re 在序列 [a,b) 内的所有匹配，并使用 match_flag_type m 来查找。 对象 re 必须在 regex_iterator 的生命周期内存在。

Throws: std::runtime_error if the complexity of matching the expression against an N character string begins to exceed O(N2), or if the program runs out of stack space while matching the expression (if Boost.Regex is configured in recursive mode), or if the matcher exhausts its permitted memory allocation (if Boost.Regex is configured in non-recursive mode).
异常：如果对于N字符长度表达式匹配的复杂度超过O(N2)，或者表达式匹配时发生栈空间溢出 (如果 Boost.Regex 设置为递归模式)，或者匹配器耗尽了所允许申请的内存(如果 Boost.Regex 设置为非递归模式)时， 会抛出 std::runtime_error 异常。 

regex_iterator(const regex_iterator& that);

Effects: constructs a copy of that.
效果：构造 that 的一份拷贝。

Postconditions: *this == that.
后验条件：*this == that. 

regex_iterator& operator=(const regex_iterator&);

Effects: sets *this equal to those in that.
效果：设置 *this 等于 that 中的东西。

Postconditions: *this == that.
后验条件：*this == that. 

bool operator==(const regex_iterator& that)const;

Effects: returns true if *this is equal to that.
效果：如果 *this 等于 that 则返回 true. 

bool operator!=(const regex_iterator&)const;

Effects: returns !(*this == that).
效果：返回 !(*this == that). 

const value_type& operator*()const;

Effects: dereferencing a regex_iterator object it yields a const reference to a match_results object, whose members are set as follows:
效果：提领一个 regex_iterator 对象，产生一个 match_results 对象的常量引用，其成员设置如下：

re.mark_count()

(*it).empty()

The end of the last match found, or the start of the underlying sequence if this is the first match enumerated
最后一次找到的匹配的结尾迭代器，如果这是第一次匹配，则为底层序列的开始迭代器

(*it).prefix().last

The same as the start of the match found: (*it)[0].first
与找到的匹配的开始迭代器相同：(*it)[0].first

(*it).suffix().matched

True if the suffix did not match an empty string: (*it).suffix().first != (*it).suffix().second
如果后缀不匹配一个空串则返回 true：(*it).suffix().first != (*it).suffix().second

(*it)[0].matched

true if a full match was found, and false if it was a partial match (found as a result of the match_partial flag being set).

(*it)[n].first

For all integers n < (*it).size(), the start of the sequence that matched sub-expression n. Alternatively, if sub-expression n did not participate in the match, then last.
对于所有整数 n < (*it).size()，为匹配的子表达式 n 的序列起始点。如果子表达式 n 不参与该匹配，则为 last。

(*it)[n].second

For all integers n < (*it).size(), the end of the sequence that matched sub-expression n. Alternatively, if sub-expression n did not participate in the match, then last.
对于所有整数 n < (*it).size()，为匹配的子表达式 n 的序列结尾点。如果子表达式 n 不参与该匹配，则为 last。

(*it)[n].matched

For all integers n < (*it).size(), true if sub-expression n participated in the match, false otherwise.
对于所有整数 n < (*it).size()，如果子表达式 n 参与该匹配，则为 true，否则为 false。

(*it).position(n)

For all integers n < (*it).size(), then the distance from the start of the underlying sequence to the start of sub-expression match n.
对于所有整数 n < (*it).size()，为从底层序列起始点到子表达式 n 的起始点的距离。 

const value_type* operator->()const;

Effects: returns &(*this).
效果：返回 &(*this). 

regex_iterator& operator++();

Effects: moves the iterator to the next match in the underlying sequence, or the end of sequence iterator if none if found. When the last match found matched a zero length string, then the regex_iterator will find the next match as follows: if there exists a non-zero length match that starts at the same location as the last one, then returns it, otherwise starts looking for the next (possibly zero length) match from one position to the right of the last match.
效果：将迭代器移到潜在序列的下次匹配，如果不存在下次匹配则为序列迭代器的结尾。 如果上次匹配零长度字符串，那么 regex_iterator 会按照如下过程找到下次匹配： 如果在上次匹配的相同位置处为起点存在非零长度的匹配，则返回它，否则从上次右边的一个位置查找下次匹配(可能为零长度)。

Throws: std::runtime_error if the complexity of matching the expression against an N character string begins to exceed O(N2), or if the program runs out of stack space while matching the expression (if Boost.Regex is configured in recursive mode), or if the matcher exhausts its permitted memory allocation (if Boost.Regex is configured in non-recursive mode).
异常：如果对于N字符长度表达式匹配的复杂度超过O(N2)，或者表达式匹配时发生栈空间溢出 (如果 Boost.Regex 设置为递归模式)，或者匹配器耗尽了所允许申请的内存(如果 Boost.Regex 设置为非递归模式)时， 会抛出 std::runtime_error 异常。

Returns: *this.
返回：*this. 

regex_iterator operator++(int);

Effects: constructs a copy result of *this, then calls ++(*this).
效果：构造 *this 的结果的一份拷贝，然后调用 ++(*this)。

Returns: result.
返回：结果。

Examples 示例

The following example takes a C++ source file and builds up an index of class names, and the location of that class in the file.
以下例子接受一个C++源文件，对其中的类名建立一个索引，以及这些类在文件中的位置。 

#include <string>
#include <map>
#include <fstream>
#include <iostream>
#include <boost/regex.hpp>

using namespace std;

// purpose:
// takes the contents of a file in the form of a string
// and searches for all the C++ class definitions, storing
// their locations in a map of strings/int's

typedef std::map<std::string, std::string::difference_type, std::less<std::string> > map_type;

const char* re = 
   // possibly leading whitespace:   
   "^[[:space:]]*" 
   // possible template declaration:
   "(template[[:space:]]*<[^;:{]+>[[:space:]]*)?"
   // class or struct:
   "(class|struct)[[:space:]]*" 
   // leading declspec macros etc:
   "("
      "\\<\\w+\\>"
      "("
         "[[:blank:]]*\\([^)]*\\)"
      ")?"
      "[[:space:]]*"
   ")*" 
   // the class name
   "(\\<\\w*\\>)[[:space:]]*" 
   // template specialisation parameters
   "(<[^;:{]+>)?[[:space:]]*"
   // terminate in { or :
   "(\\{|:[^;\\{()]*\\{)";


boost::regex expression(re);
map_type class_index;

bool regex_callback(const boost::match_results<std::string::const_iterator>& what)
{
   // what[0] contains the whole string
   // what[5] contains the class name.
   // what[6] contains the template specialisation if any.
   // add class name and position to map:
   class_index[what[5].str() + what[6].str()] = what.position(5);
   return true;
}

void load_file(std::string& s, std::istream& is)
{
   s.erase();
   s.reserve(is.rdbuf()->in_avail());
   char c;
   while(is.get(c))
   {
      if(s.capacity() == s.size())
         s.reserve(s.capacity() * 3);
      s.append(1, c);
   }
}

int main(int argc, const char** argv)
{
   std::string text;
   for(int i = 1; i < argc; ++i)
   {
      cout << "Processing file " << argv[i] << endl;
      std::ifstream fs(argv[i]);
      load_file(text, fs);
      // construct our iterators:
      boost::sregex_iterator m1(text.begin(), text.end(), expression);
      boost::sregex_iterator m2;
      std::for_each(m1, m2, &regex_callback);
      // copy results:
      cout << class_index.size() << " matches found" << endl;
      map_type::iterator c, d;
      c = class_index.begin();
      d = class_index.end();
      while(c != d)
      {
         cout << "class \"" << (*c).first << "\" found at index: " << (*c).second << endl;
         ++c;
      }
      class_index.erase(class_index.begin(), class_index.end());
   }
   return 0;
}

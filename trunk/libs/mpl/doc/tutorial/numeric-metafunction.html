数值元函数协议/基础架构

算术、比较和位操作元函数现在都是多态的，可以操作于多种数字类型，包括有理数、定点数和复数。它们允许混合算术，即你可以对不同类型的参数进行操作，结果将是最大/最通用的参数类型。这一架构允许用户定义的数字类型可以自由地与预定义的类型进行混合操作。例子请见 http://cvs.sourceforge.net/viewcvs.py/boost/boost/libs/mpl/test/numeric_ops.cpp?view=markup, 正式的架构说明请见 参考手册。

如果你在将 MPL 数字元函数用于你自己的整型包装类，如 mpl::int_, 你可以将你的类加到这个新架构中，方法是用以下成员扩展它：

typedef mpl::integral_c_tag tag;

例如：

以前 现在 
template< int n > struct my_int
{
    static int const value = n;
    typedef my_int type;
};
 
template< int n > struct my_int
{
    typedef mpl::integral_c_tag tag;
    static int const value = n;
    typedef my_int type;
};

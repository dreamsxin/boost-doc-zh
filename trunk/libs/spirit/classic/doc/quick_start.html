快速上手

你为什么想用 Spirit ?

Spirit 设计为实际的解析工具。 从一个内联于C++中的标准EBNF规范，产生一个能完全工作的解析器，这种能力的确能减少了开发的时间。当要开发一 个计算机语言例如C或Pascal，实际中也可用一个成熟并且卓越的解析器（如YACC，ANTLR），但是当我们需要非常微小的解析器时，继续这样做无 异于大炮打蚊子。在这领域的一端，程序员在着手处理时一般不是把任务看待为标准的解析任务，而是特定地用粗糙的工具如scanf来"搞"任务。 真的，是有一些解析工具例如正则表达式库 (如 boost regex) 或者扫描仪 (如 boost tokenizer)， 但是当我们需要写更精细的解析器的时候，这些工具不能很好地伸缩扩展。 如果企图用这些工具写一个适度复杂的解析器，将导致难理解，难维护的代码。

其中一个最初目的是使工具很容易使用。 当提起解析器生成器时，通常的想法是“它一定是很大很复杂，并且很难学的”。 现在不是这样了！ Spirit 设计成完全可伸缩。 Spirit的框架是层次化结构。 在学习最小的内核和基础概念之后，可以只学习需要的基础。

为了开发的简易，整个框架只由头文件组成，不需要连接或者编译额外的库。 只需要把Spirit的发布包放在include目录下，编译，运就可以了。 代码的大小？非常少。在快速上手中，我们给出很简短的例子，代码的大小主要由std::vector和std::iostream实例化的开销主导。

简单例子 #1

建立一个解析浮点数的解析器。 

    real_p

(你必须承认，这很简单！) 上面的代码产生一个内置的浮点数解析器 real_parser 。 要注意，用户可以直接用的解析器以"_p"结尾，这是个Spirit的习惯约定。Spirit有很多预定义的解析器，有一致的命名约定，那你就不会疯掉了！

简单例子 #2

建立一个解析器，接受由两个浮点数组成的一行。 

    real_p >> real_p

这里你看到熟悉的浮点数解析器 real_p 被用了两次， 每个数用一次。 那么操作符 >> 在这里做什么呢？ 恩，一定要有些东西分开它们吧，操作符>>用作“紧跟着”序列操作符。 上面的程序通过用胶水(序列操作符)把两个简单的解析器粘在一起，建立了一个稍复杂的解析器。  数字之间的空白符的处理隐含于解析器的调用方法（见下面）。

注意：对解析器进行组合的时候，我们得到一个“更大”的解析器，但是它仍然是个解析器，所以可以继续嵌套，越变越大。总之，当你用胶水的时候，将产生一个更大的解析器，这是一个重要的概念。

简单例子 #3

建立一个解析器，它能够接受任意个数的浮点数。（任意的意思是从零到无穷大）

    *real_p

这样很像正则表达式的 Kleene 星号，语法上可能有点古怪，特别对于那些没见过重载*操作符的C++程序员。 但事实上，如果你了解过正则表达式，古怪之处或许在于星号是作为表达式的前缀而不是后缀。 C'est la vie先生把这种形式不协调指责于我们不得不遵守的C++语法规则。

Kleene 星号可用在任意的解析器表达式上。 但要谨记着，因为C++操作符的优先级规则，你可能需要用圆括号括上复杂的表达式。  Kleene 星号也被称作 Kleene 结尾，但文中多数地方仍称之为 Kleene星号。

例子 #4 [ 一个稍不简单的例子 ]

这个例子将建立一个解析器，接受一串用逗号分隔的浮点数。

步骤 1. 建立解析器

    real_p >> *(ch_p(',') >> real_p)

请注意 ch_p(',') 。 这是一个识别逗号','的字面字符解析器。 在这里，Kleene星号作用在一个更复杂的解析器上，这是指由下面的表达式生成的解析器： 

    (ch_p(',') >> real_p)

要注意，这种情况必须要用圆括号。 Kleene 星号作用在上面写的整个表达式上。

步骤 2. 使用解析器 (在建立之后)

既然已经建立了一个解析器，那怎么用它呢？ 就像任意的C++临时对象一样，我们可以把它存在一个变量里面，或者直接在上面调用函数。

我们忽略掉C++的底层细节，仅仅给出精彩的部分。

设 r 是一条规则（不要在意到底什么是规则，以后就会讨论清楚。在这里只需知道，规则是一个可以存储解析器的占位变量。） ， 那么，我们按如下方法以规则的形式存储一个解析器： 

    r = real_p >> *(ch_p(',') >> real_p);

并不十分惊讶，这只是你用了多年的平常的附值操作。 真正酷的事情是，当你把解析器存储在规则中时，规则就是解析器，以后你就可以用名字来引用它。 (在上面情况下，规则名字就是 r)。 要注意，这是一个完整的附值表达式，所以我们要用分号";"终结。

正是如此。我们已经懂得去定义一个解析器了。那么下一步就是要让解析器进行操作了。 这有很多方法。 在这里，我们介绍的方法是用非成员函数 parse 来解析一个字符串 char const* 。 这个函数有三个输入参数：

以带零终止符的 const char* 作为输入字符串
解析器对象
另外一个称为 skip parser（跳读解析器）的解析器

在如下例子中，我们希望跳过空白符和tab符。 另一个称为space_p的解析器是Spirit库预定义的。一个非常简单的只识别空白符的解析器。 我们会用 space_p 作为跳读解析器。 跳读解析器负责跳过那些解析器基元（如real_p和ch_p）之间的字符。

好了，让我们开始解析吧！ 

    r = real_p >> *(ch_p(',') >> real_p);
    parse(str, r, space_p) // 还不是完成的表述，请继续耐心阅读...

函数 parse 返回一个叫作parse_info的对象，它拥有解析的结果和其他信息。在这个例子中，我们想知道：

解析器是否成功识别输入字符串 str ？
解析器是否完全解析和处理了整个字符串？

为了清楚目前整个情况，我们把解析器包装在一个函数中： 

    bool
    parse_numbers(char const* str)
    {
        return parse(str, real_p >> *(',' >> real_p), space_p).full;
    }

这里要注意我们没有用带名字的规则，而直接把解析器内联在parse函数中。当调用函数parse时，表达式的值被放在一个临时解析器变量中，这个无名的解析器被传入到parse函数，用完后就销毁。

char 和 wchar_t 操作数

细心的读者可能注意到解析器表达式里用一个','代替了之前例子的ch_p(',')。 这在C++的类型转换语法规则中是允许的。 库中重载了>>操作符，接受char或者wchar_t类型的参数，左参数或者右参数（但不能同时）。 起码一个参数是用户定义类型的时候，操作符就会进行重载。 在例子的情况下， real_p 是operator>>的第二个参数，所以重载了恰当的>> ， ',' 被自动转换为字面字符解析器。

省略ch_p 会有明显的问题： 'a' >> 'b' 不是一个spirit解析器，而是一个数值表达式，代表对ASCII值（或其他编码）'a' 进行右移ASCII值'b'的操作。 但是， ch_p('a') >> 'b' 和 'a' >> ch_p('b') 都是 Spirit方式的序列解析器，字 'a' 后面紧跟着'b'。你迟早会习惯这样的。

请注意，从parse函数返回的对象有一个布尔成员叫full，它是真（true），仅当我们的两个要求都满足（也就是输入字符串被完全地解析）。

步骤 3. 语义行为

刚才的解析器只不过是一个识别器。它回答了问题"输入符合我们定义的文法吗？"， 但它没有记录任何数据，也没有执行额外的操作。 谨记：我们想把被解析的数字放入一个vector中。 这通过一种action操作来完成，此操作要连接到特定的解析器中。 例如，当我们解析一个实数的时候，我们希望保存下成功解析匹配得到的数。 我们现在希望从解析器中提取信息。 这需要执行语义行为。 语义行为可以连接在语法规范的任意一点上。 这些行为是C++的函数或者函数子(functor)；当解析器的一部分成功匹配了输入的一部分时，就会调用语义行为。 设你有一个解析器P，和一个C++函数 F，你可以把F连接到解析器，当解析器匹配了输入时则会调用F： 

    P[&F]

又或者 F 是一个函数子（functor）时： 

    P[F]

这种函数/函数子签名，依赖于它连接的解析器的类型。 解析器 real_p 传递单一个参数：被解析的数。因此如果我们要连接函数F到real_p ，我们要求F被声明为： 

    void F(double n);

在我们的例子中，我们再次利用一些预定义的语义函数子和函数子生成器（一个函数子生成器是一个返回函数子的函数）。 为了达到目的，我们用Spirit的一个函数子生成器 push_back_a(c)。 简单地讲，这个语义行为被调用时，会把从连接的解析器接收到的的数appends添加到一个容器c中。

最终，这是我们完整的解析器，可以处理逗号分隔的浮点数列表： 

    bool
    parse_numbers(char const* str, vector<double>& v)
    {
        return parse(str,

            //  文法开始
            (
                real_p[push_back_a(v)] >> *(',' >> real_p[push_back_a(v)])
            )
            ,
            //  文法结束

            space_p).full;
    }

这是和之前相同的解析器。这次有合适的语义行为连接到计划好的地方，这样解析到的数会被提取出来填充在vector v中。 如果解析成功，函数 parse_numbers 返回真。

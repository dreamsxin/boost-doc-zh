Boost Exception

current_exception

#include <boost/exception_ptr.hpp>

namespace
boost
    {
    exception_ptr current_exception();
    }

要求:

在catch块的外部不能在调用函数 current_exception.

返回值:

  • 异常对象指针 exception_ptr ,指向当前处理的异常对象或者其拷贝.
  • 如果函数需要分配内存,但是分配失败, exception_ptr 对象指向一个 std::bad_alloc 的实例.

异常抛出:

没有异常.

注意:

  • 连续两次的调用函数 current_exception ,并不保证得到的总是同一的异常对象.
  • 在需要抛出当前要处理的异常的时候,除非调用了函数enable_current_exception, 否则函数 current_exception 的实现可能需要编译器的支持. 如果没有调用函数 enable_current_exception , 并且编译器没有提供必要的支持,那么函数 current_exception 可能返回一个指向unknown_exception类型的异常对象指针. 在这种情况下, 如果初始的异常对象是从 boost::exception继承而来, 那么通过boost::exception 的copy构造函数, unknown_exception 类型的子对象就会被构造出来.

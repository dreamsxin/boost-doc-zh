线程之间传递异常

Boost 异常通过clone 支持线程之间的异常传递. 这跟 N2179是相似的, 但是因为 Boost 异常不能仅仅依赖于语言的支持, 这就需要在抛出异常的时候使用函数 enable_current_exception 对异常对象进行clone.

注意:

所有通过函数 boost::throw_exception 抛出的异常都是从 boost::exception 继承的,并且支持clone.

抛出异常的时候使用 enable_current_exception

下面的例子说明如何在throw语句中支持clone (15.1):

#include <boost/exception/info.hpp>
#include <stdio.h>
#include <errno.h>

typedef boost::error_info<struct tag_errno,int> errno_info;

class file_read_error: public boost::exception { };

void
file_read( FILE * f, void * buffer, size_t size )
    {
    if( size!=fread(buffer,1,size,f) )
        throw boost::enable_current_exception(file_read_error()) <<
            errno_info(errno);
    }

当然, 函数enable_current_exception 支持任何的异常类型,没有要求说异常类型一定要从boost::exception继承.

clone并再次抛出异常对象

当你捕获异常对象, 你可以通过函数 current_exception 得到 exception_ptr 得到它:

#include <boost/exception_ptr.hpp>
#include <boost/thread.hpp>
#include <boost/bind.hpp>

void do_work(); //throws cloning-enabled boost::exceptions

void
worker_thread( boost::exception_ptr & error )
    {
    try
        {
        do_work();
        error = boost::exception_ptr();
        }
    catch(
    ... )
        {
        error = boost::current_exception();
        }
    }

在上面的实例中, 注意 current_exception 捕获的是原始类型的异常对象, 并且可以通过函数 rethrow_exception 重新抛出:

// ...continued

void
work()
    {
    boost::exception_ptr error;
    boost::thread t( boost::bind(worker_thread,boost::ref(error)) );
    t.join();
    if( error )
        boost::rethrow_exception(error);
    }

注意函数 current_exception 会在下面的情形中导致不能拷贝原始的异常对象 :

  • 没有足够的空间, 在那种情况下指针 exception_ptr 指向std std::bad_alloc类型的异常, 或者
  • 如果在throw 语句中没有使用函数 enable_current_exception ,并且当前的编译器不支持自动拷贝异常对象, 那么得到的指针exception_ptr 指向 unknown_exception类型的对象.

无论如何, 在上面的实例中,函数 current_exception 和 rethrow_exception 的使用在形式上都是合法的.

Design Rationale（设计原则）

有一个重要的关于构造函数的设计权衡：我们可以将 array 实现为一个“集合体”（参见 C++ 标准的 8.5.1, [dcl.init.aggr]）。这就意味着：

  • 一个数组能被一个大括号括起来的，逗号分隔的初始化列表初始化，初始化列表中的值按照容器中各元素的下标递增的顺序排列：

    boost::array<int, 4> a = { { 1, 2, 3 } };

    注意，如果初始化列表中的值少于元素的个数，则多余的元素会被默认初始化（这样，它就有了一个已定义的值）。

然而，这种方法也有它的劣势：不传递初始化列表意味着元素有一个不确定的初始值，因为规则规定集合体应该：

  • 无用户声明的构造函数。
  • 无私有的或保护的非静态数据成员。
  • 无基类。
  • 无虚拟函数。

无论如何，当前的实现使用了这个方法。

注意，对于与标准兼容的编译器，可能会使用更少的大括号（根据标准的 8.5.1 (11)）。也就是说，你可以像这样初始化一个 array：

boost::array<int, 4> a = { 1, 2, 3 };

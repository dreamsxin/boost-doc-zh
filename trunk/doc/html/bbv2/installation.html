Installation 安装

This section describes how to install Boost.Build from a released Boost source distribution or CVS image. [4] All paths are given relative to the Boost.Build v2 root directory, which is located in the tools/build/v2 subdirectory of a full Boost distribution. [5]
本节描述如何从一个 Boost 源分发包 或 CVS 映像 安装  Boost.Build。[4] 所有给出的路径均相对于 Boost.Build v2 的根目录，位于完整的 Boost 分发包 的 tools/build/v2 子目录。[5]

  1. Boost.Build uses Boost.Jam, an extension of the Perforce Jam portable make replacement. The recommended way to get Boost.Jam is to download a prebuilt executable from SourceForge. If a prebuilt executable is not provided for your platform or you are using Boost's sources in an unreleased state, it may be necessary to build bjam from sources included in the Boost source tree.
    Boost.Build 使用了 Boost.Jam，它是 Perforce Jam 的扩展，一个可移植的 make 替代品。获取 Boost.Jam 的推荐方式是从 SourceForge 下载一个预构建的可执行文件。 如果没有为你的平台提供一个预构建的可执行文件，或者你使用的是处于未发布状态的 Boost 源文件，就可能需要用包含在 Boost 源文件树中的 源程序来构建 bjam。

  2. To install Boost.Jam, copy the executable, called bjam or bjam.exe to a location accessible in your PATH. Go to the Boost.Build root directory and run bjam --version. You should see:
    要安装 Boost.Jam，请将名为 bjam 或 bjam.exe 的可执行文件拷贝到你的 PATH 中的一个可访问位置。转到 Boost.Build 的根目录并运行 bjam --version. 你将看到： 

                Boost.Build V2 (Milestone N)
                Boost.Jam xx.xx.xx

    where N is the version of Boost.Build you're using.
    其中 N 为你正在使用的 Boost.Build 的版本。

  3. Configure Boost.Build to recognize the build resources (such as compilers and libraries) you have installed on your system. Open the user-config.jam file in the Boost.Build root directory and follow the instructions there to describe your toolsets and libraries, and, if necessary, where they are located.
    配置 Boost.Build 以认可你的系统中已安装的构建资源(如编译器和库)。打开位于 Boost.Build 根目录中的 user-config.jam 文件并按照那里的指令描述你的工具集和库，并且，如果需要，还要给出它们的位置。
  4. You should now be able to go to the example/hello/ directory and run bjam there. A simple application will be built. You can also play with other projects in the example/ directory.
    你现在应该可以转到 example/hello/ 目录并在那里运行 bjam。一个简单的应用程序将被构建。你也可以试一下在 example/ 目录下的其它工程。

If you are using Boost's CVS state, be sure to rebuild bjam even if you have a previous version. The CVS version of Boost.Build requires the CVS version of Boost.Jam.
如果你使用 Boost 的 CVS 状态，请确认即使你已有一个旧的版本也要重新构建 bjam。Boost.Build 的 CVS 版本需要 Boost.Jam 的 CVS 版本。

When bjam is invoked, it always needs to be able to find the Boost.Build root directory, where the interpreted source code of Boost.Build is located. There are two ways to tell bjam about the root directory:
当 bjam 被调用时，必须总是要能够找到 Boost.Build 根目录，以解释 Boost.Build 的源代码位置。有两种方法告诉 bjam 根目录在哪：

  • Set the environment variable BOOST_BUILD_PATH to the absolute path of the Boost.Build root directory.
    设置环境变量 BOOST_BUILD_PATH 为 Boost.Build 根目录的绝对路径。

  • At the root directory of your project or in any of its parent directories, create a file called boost-build.jam, with a single line:
    在你的工程的根目录或它的任一父目录，创建一个名为 boost-build.jam 的文件，其中只有一行： 

    boost-build /path/to/boost.build ;

Information for distributors 发行人的信息

If you're planning to package Boost.Build for a Linux distribution, please follow these guidelines:
如果你计划为一个 Linux 分发包打包 Boost.Build，请按以下指引：

  • Create a separate package for Boost.Jam.
    为 Boost.Jam 创建一个独立包。

  • Create another package for Boost.Build, and make this package install all Boost.Build files to /usr/share/boost-build directory. After install, that directory should contain everything you see in Boost.Build release package, except for jam_src directory. If you're using Boost CVS to obtain Boost.Build, as opposed to release package, take everything from the tools/build/v2 directory. For a check, make sure that /usr/share/boost-build/boost-build.jam is installed.
    为 Boost.Build 创建另一个包，并让这个包将所有 Boost.Build 文件安装到 /usr/share/boost-build 目录。在安装后，该目录应包含你在 Boost.Build 发布包中看到所有东西，除了 jam_src 目录。如果你正在使用 Boost CVS 来得到 Boost.Build 的，与发布包相反，应从 tools/build/v2 目录获取所有东西。作为一个检查，请确认 /usr/share/boost-build/boost-build.jam 已安装。

    Placing Boost.Build into /usr/share/boost-build will make sure that bjam will find Boost.Build without any additional setup.
    将 Boost.Build 放入 /usr/share/boost-build 可以确保 bjam 无需任何额外设置就可以找到 Boost.Build。

  • Provide a /etc/site-config.jam configuration file that will contain:
    提供一个 /etc/site-config.jam 配置文件，其中应包含： 

    using gcc ;

    You might want to add dependency from Boost.Build package to gcc, to make sure that users can always build Boost.Build examples.
    你也许想为 Boost.Build 包增加到 gcc 的依赖关系，以确保用户总是可以构建 Boost.Build 例子。

If those guidelines are met, users will be able to invoke bjam without any explicit configuration.
只要按照以上指引，用户无需进行任何配置就可以调用 bjam。


[4] Note that packages prepared for Unix/Linux systems usually make their own choices about where to put things and even which parts of Boost to include. When we say “released source distribution” we mean a distribution of Boost as released on its SourceForge project page.
注意，为 Unix/Linux 系统准备的包，对于将东西放在何处，甚至包含 Boost 的哪部分，通常有它们自己的选择。我们所说的"发布版本源文件包"是指在 Boost 的 SourceForge 工程主页 上发布的 Boost 发布包。 

[5] The Boost.Build subset of boost is also distributed separately, for those who are only interested in getting a build tool. The top-level directory of a Boost.Build distribution contains all the subdirectories of the tools/build/v2 subdirectory from a full Boost distribution, so it is itself a valid Boost.Build root directory. It also contains the tools/jam/src subdirectory of a full Boost distribution, so you can rebuild Boost.Jam from source.
对于那些只对构建工具感兴趣的用户，boost 的 Boost.Build 子集也是独立分发的。Boost.Build 分发包 的顶层目录包含了一个完整的 Boost 分发包中的 tools/build/v2 子目录下的所有子目录，所以它本身就是一个有效的 Boost.Build 根目录。它还包含了一个完整的 Boost 分发包中的 tools/jam/src 子目录，所以你也可以从源文件重新构建 Boost.Jam。

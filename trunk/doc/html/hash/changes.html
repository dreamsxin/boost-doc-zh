Change Log（变化日志）

Boost 1.33.0

  • 最初版本

Boost 1.33.1

  • 修复由沈慧峰指出的指针实例。

Boost 1.34.0

  • 为标准类使用声明，以便库不再需要包含所有的头文件。
  • 不赞成使用 <boost/functional/hash/*.hpp> 头文件。使用单独的头文件 <boost/functional/hash.hpp>
  • 增加对 BOOST_HASH_NO_EXTENSIONS 宏的支持，它可以使对 TR1 的扩展无效。
  • 浮点数的散列函数的小幅度改进。
  • 更新可移植的示例，但愿能更广泛地一致。

Boost 1.34.1

  • Ticket 952：消除 Visaul C++ 上不正确的 64 位警告。

Boost 1.35.0

  • 为 long long, std::complex 提供支持。
  • 改进散列浮点数的算法：
    • 改进可移植性，就像 Daniel Krügler 在 a post to the boost users list 中描述的一样。
    • 在每次结合循环中装入更多信息，这样就能减少调用结合的次数并有望得到一个品质更好的散列函数。
    • 改进散列浮点数的算法。
    • 在 Cygwin 中为浮点数使用一个二元的散列函数，因为 Cygwin 没有适宜的针对 long double 的浮点机能。
    • 不再使用不支持 long double 的 fpclass。
    • Ticket 1064：移除了不必要的 errno 的使用。
  • 针对更多内建类型的显示重载。
  • 文档中的小规模改进。
  • 修改少数 bug 和警告：
    • Ticket 1509：消除其它 Visual C++ 警告。
    • 针对 Sun 编译器的一些 workarounds。

Boost 1.36.0

  • 停止使用 OpenBSD 的不可靠的 std::numeric_limits。
  • 为 long long 和 unsigned long long 使用 boost typedefs。
  • 将扩展移入它们自己的头文件。

Boost 1.37.0

  • Ticket 2264: 在 Visual C++ 中，对于 long double 和 float 总是使用C99浮点函数，因为C++重载不总是可用的。

Boost 1.38.0

  • 将1.34.0起不再使用的头文件，由告警改为错误。在未来的Boost版本中，这些头文件将被去掉。
  • 将实现细节的头文件移出 boost/functional/detail，因为它们是 functional/hash 而不是 functional 的一部分。 boost/functional/detail/container_fwd.hpp 被移至 boost/detail/container_fwd.hpp，由于它在本库之外使用，其它文件则被移至 boost/functional/hash/detail.
